# 品質チェックリスト（リリース前 / 大きな変更後）

Fable5 が最終確認する項目。

## ビルド・型
- [ ] `cargo check` がエラー 0
- [ ] `npx tsc --noEmit` がエラー 0
- [ ] `npm run tauri dev` で起動し、2 キャラクターが表示される

## 透過ウインドウ / クリック透過（spec 3.1）
- [ ] キャラクター上のクリックが反応する（ドラッグで移動できる）
- [ ] 透明部分のクリックが背後のデスクトップに抜ける
- [ ] クリック透過状態から不透明部分に戻ったとき操作が復帰する（再入検知）
- [ ] セーフモード（枠あり通常ウインドウ）に切り替えられる

## 表情切り替え（spec 3.2）
- [ ] 全表情がプリロードされ、切り替え時にチラつき・白矩形が出ない

## 対話エンジン
- [ ] 低負荷モードで YAML 辞書のキーワードマッチが動く
- [ ] アドヴァンスモードで main/sub の掛け合いがシングルコールで返る
- [ ] LLM 出力の JSON パース失敗時にフォールバック台詞が出る（spec 3.3）
- [ ] API エラー（401/429/timeout）で低負荷モードに自動フォールバックし、専用台詞が出る
- [ ] モード切替で文脈が引き継がれる（低→アド: 直近ログ挿入 / アド→低: 要約生成）

## 独り言キャッシュ（spec 2.4）
- [ ] 対話のついでに monologues がキャッシュされる
- [ ] ランダムトークが API コールなしでキャッシュから消費される
- [ ] キャッシュ枯渇時は辞書の random_talk にフォールバックする

## コスト・セキュリティ（spec 2.5）
- [ ] API キーが keyring（OS 資格情報マネージャー）に保存され、平文ファイルに残らない
- [ ] 今月の推定消費額が設定画面に表示される
- [ ] 上限到達で低負荷モードに強制フォールバックする

## 互換性・拡張（spec 2.6）
- [ ] ghost.json / shell.json / dic/*.yaml に schema_version がある
- [ ] 未知の新しい schema_version を読んだとき、クラッシュせず警告して拒否する
- [ ] TTS フック（イベント/トレイト）が対話出力経路に存在する

## 非同期（spec 3.4）
- [ ] 要約・独り言生成がメイン応答の描画をブロックしない（busy ゲートで待機する）

## 常駐基盤（spec 5.1）
- [ ] トレイアイコンが表示され、左クリックで表示/非表示が切り替わる
- [ ] トレイからモード・静音・セーフモードを切り替えると設定パネルの表示も追従する
- [ ] トレイ「終了」で終了挨拶が出てからアプリが終了する（静音中は即終了）
- [ ] ウインドウを移動して再起動すると同じ位置に出る
- [ ] モニタ外の保存位置では中央にフォールバックする
- [ ] セーフモードの×ボタンで終了せず非表示になる（トレイから復帰できる）
- [ ] 自動起動トグルで OS のスタートアップ登録が増減する
- [ ] 初回起動時のみオンボーディング台詞（first_boot）が出る
- [ ] セーフモード切替（ページ再ロード）で挨拶が二重に出ない

## イベント対話（spec 5.2）
- [ ] 起動挨拶が時間帯（朝/昼/夜）で変わる
- [ ] キャラのダブルクリックでつつき反応、4 連打で連打反応が出る
- [ ] クリック 1 回では従来どおり入力欄がトグルする
- [ ] 30 分無操作で idle 台詞が 1 回だけ出る（操作でリセット）
- [ ] schema_version: 1 の旧辞書も警告なしで読める（events 無しとして動く）

## 長期記憶（spec 5.3）
- [ ] 会話で名前を教えると user_profile に保存され、以降の応答に反映される
- [ ] 設定パネルで記憶の閲覧・手動追加・個別削除ができる

## 配慮・運用（spec 5.4）
- [ ] 静音モード中はランダムトーク・idle が止まり、話しかけには返事する
- [ ] 全画面アプリ使用中はランダムトークが止まる（auto_quiet_fullscreen）
- [ ] ウインドウ非表示中はランダムトークが止まる
- [ ] 履歴削除は 2 段階確認で、「記憶も消す」チェックが機能する
- [ ] update_feed_url 設定時、新バージョンで一度だけ更新通知台詞が出る

## 時事ネタ・興味分野・オンボーディング（spec 5.5）
- [ ] 初回起動でプロフィール聞き取りパネルが出る（スキップ可、設定の「プロフィールを設定し直す」で再表示）
- [ ] 興味分野を登録し「今すぐ取得」で見出しが集まり、高度モードの独り言に時事ネタが混ざる
- [ ] 暗い見出し（事故・災害等）が選ばれない／低負荷モードでは見出しを発話しない
- [ ] 既定オフで、明示的にオンにしたときだけ web 取得が動く

## 音声合成 TTS（spec 5.6）
- [ ] 既定エンジン = 埋め込み(voicevox_core)。**別アプリ/サーバ不要**で「声のテスト」が鳴る
- [ ] 初回は資産未取得状態が「未取得です」と表示され、規約同意 → DL → 「✓ 準備OK」になる
- [ ] DL 失敗時はエラー文と最後の進捗が画面に残り、再試行できる（一瞬で消えない）
- [ ] GitHub レート制限時は日本語の案内 + PAT 入力欄が機能。PAT 保存→再起動後も自動利用
- [ ] **クレジット表記「VOICEVOX:話者名」が画面下端中央に常時表示**（声/エンジン変更で追従、TTS無効/openai_compat では非表示）
- [ ] 話者ドロップダウンに同じ声が**重複表示されない**（埋め込み・HTTP 両方）
- [ ] boot 直後・設定変更時にバックグラウンドで事前 init が走り、初回発話のラグが消える
- [ ] 有効化すると応答・独り言・つつき・撫で・時事ネタすべてに声が乗る
- [ ] 無操作の独り言でも声が鳴り続ける（autoplay 対応・キュー固着しない）
- [ ] 資産未DL or 外部エンジン未起動でも声なしで会話が継続し、アプリが落ちない
- [ ] エンジン切替（core ↔ http ↔ openai_compat）で UI のフィールド表示が正しく出し分けされる
- [ ] OpenAI互換選択時、声欄のキャプション（VoiceDesign）が `instructions` として送られる（参照音声クローンは未使用）

## 音声入力 STT（spec 5.6・現在は無効化）
- [ ] 設定に「音声入力」セクションが表示されない／入力欄にマイクボタンが出ない（STT_UI_ENABLED=false）

## ツール（spec 5.6）
- [ ] tools_enabled で「今何時?」「今日何曜日?」に答える
- [ ] 「N 分後に教えて」でリマインダーが時間どおり発話する（静音中も鳴る）
- [ ] 📋 ボタンを押して送ると、クリップボードの内容をふまえて翻訳/要約する（押さないときは読まない）
- [ ] tools 無効時は一切挙動が変わらない

## 対話・設定 UX（spec 5.7）
- [ ] 設定上部のカテゴリ・プルダウンで 6 カテゴリを切り替えられ、全項目に到達できる
- [ ] 設定を保存すると設定画面が自動で閉じる
- [ ] 長いセリフほど長く表示され、TTS 発話中は消えない／メイン・サブが出そろってから一括消去される
- [ ] 高度モードで掛け合いの順番が変わる（サブ先行のとき内容もサブ先行になっている）
- [ ] パターン 3/4 で 3 ターン目が最初の話者の吹き出しに 1 行空けて追記される
- [ ] 低負荷モード・独り言は常にメイン先行（パターン 1）

## 口パク（spec 5.8）
- [ ] `<表情>_talk.png` を持つデフォルトシェルで、TTS 読み上げ中に口が動く（話しているキャラのみ）
- [ ] 発話が終わると口が閉じる／TTS 無効時はタイプライター中に口が動く
- [ ] 開口フレームが無いシェルでは口パクしない（従来どおり・エラーなし）

## ポモドーロ（spec 5.9）
- [ ] 設定「会話・ふるまい」or トレイから開始でき、上部にカウントダウンバッジが出る
- [ ] 集中中はランダムトーク・放置反応が止まる／停止・全完了で雑談に戻る
- [ ] 集中終了・休憩終了・全完了の節目でキャラが声をかける（静音中でも鳴る）
- [ ] バッジクリック／停止ボタン／トレイ停止で中断でき、集中モードが解除される
- [ ] 「開始」を押すと、その時点の集中/休憩/回数の入力値で始まる（未保存でも反映）

## リリース基盤（配布・運用）
- [ ] `npm run tauri build` が成功し NSIS インストーラが生成される
- [ ] インストール版を起動するとデフォルトゴーストが正しく表示される（%APPDATA%/ugga へ展開される）
- [ ] %APPDATA%/ugga/ghosts を編集 → 設定の「再読み込み」で反映される
- [ ] 起動中にもう一度起動すると、新ウインドウではなく既存ウインドウが前面に出る（二重起動防止）
- [ ] ログがファイル出力される（設定「ログフォルダを開く」で確認、5MB でローテート）
- [ ] パニック発生時にログへ記録される
- [ ] 設定のゴースト/シェルのドロップダウンに導入済みのものが並び、切替で見た目が変わる
- [ ] 表示スケールのスライダーで全体サイズとウインドウが同率で変わる（比率不変・クリック判定も追従）
- [ ] 「データを書き出す」で JSON がダウンロードフォルダに保存される（記憶チェックの有無が反映、APIキー・設定は含まない）
- [ ] 起動時 WARN/ERROR が出ない（autostart 整合の冪等化を確認）
