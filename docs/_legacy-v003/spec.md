# 開発指示書（要件の正本）

デスクトップ常駐型コンパニオンアプリ。「伺か」コンセプトを現代技術で再構築。
メインとサブの 2 体のキャラクター（エージェント）がデスクトップ上で対話し、ユーザーとコミュニケーションする。

## 技術スタック
- コア: Tauri (Rust / HTML/CSS/TypeScript)
- バックエンド: Rust (Tokio 非同期)
- フロントエンド: HTML/CSS/TypeScript (Vanilla)
- データ保存: SQLite
- 外部 API: 各種 LLM API (OpenAI, Anthropic, Grok, Ollama 等)

## 2. システム基本仕様

### 2.1 UI/UX・マルチプラットフォーム
- 2 体のキャラクターは透過 PNG/WebP の静止画。表情差分の切り替えで感情表現（連続アニメーションなし）
- デスクトップ最前面、ウインドウ枠なし（透過）。OS 別の透過処理差異は Rust 側の抽象レイヤーで吸収
- セーフモード: 不具合時のフォールバックとして枠線ありの通常ウインドウ描画モード

### 2.2 対話モード
- 低負荷モード: ローカル YAML 辞書によるパターンマッチング応答
- アドヴァンスモード: LLM 対話。シングルコール方式で 1 回の呼び出しでメイン/サブの掛け合いを同時生成
- 出力 JSON: `{"main":{"text":"...","pose":"happy"},"sub":{"text":"...","pose":"normal"}}`

### 2.3 コンテキスト維持とモード移行
- 全会話履歴（両モード）を共通 SQLite DB に保存
- 低負荷→アド移行時: 直近数件の対話ログを LLM コンテキストに挿入
- アド→低負荷移行時: 直近 LLM 対話の要約をバックグラウンド生成して SQLite に保存し、低負荷モードの辞書マッチングに反映

### 2.4 独り言（ランダムトーク）キャッシュ
- アクティブ対話中に、API リクエストのついで/バックグラウンドで「次回用の独り言ストック（掛け合い）」を生成
- SQLite にキャッシュし、タイマー起動のランダムトークはリアルタイム API コールせずキャッシュから消費

### 2.5 コスト管理・セキュリティ
- API キーは keyring クレートで OS 標準の資格情報マネージャーに保存
- 設定画面に「今月の推定 API 消費額」を表示。上限到達で低負荷モードへ自動フォールバック
- システムプロンプトでセリフ文字数を厳しく制限しトークン消費を抑制

### 2.6 ゴースト・シェルの拡張性と規約
- ゴースト（プロンプト/辞書）とシェル（画像/定義）は所定ディレクトリ構造でユーザーが追加・編集可能
- 全定義ファイルに schema_version 必須。本体更新後も旧バージョンのゴーストが動く互換処理
- 読み込みファイルの検閲・サンドボックスは行わない（非商用・自己責任の初期割り切り）
- 将来の TTS (VOICEVOX 等) を見据え、音声生成トリガーのインターフェースを対話出力処理に定義しておく

## 3. 技術的課題と対策（実装必須）

### 3.1 クリック透過制御
- 透明部分のクリックは背後に抜けさせる。ポインター直下ピクセルの Alpha 値を取得し、しきい値以下なら `set_ignore_cursor_events(true)`、不透明上では false に戻す
- 注意: ignore 中はウインドウがマウスイベントを受け取れないため、再入検知は Rust 側のグローバルカーソル位置ポーリングで行う（実装詳細は architecture.md）

### 3.2 表情切り替えのチラつき防止
- シェル画像はアプリ起動時/ゴースト変更時にメモリへプリロード
- 全表情の要素を非表示で生成しておき、切り替えはクラス付け替えのみ

### 3.3 API エラー時のフェイルセーフ
- Timeout / Unauthorized / RateLimit 等を検知したら一時的に低負荷モードへ自動切替
- エラーハンドリング専用台詞（例:「ごめん、ちょっと頭がぼーっとして通信できないみたい」）を辞書に事前定義して再生

### 3.4 バックグラウンドタスクの非同期制御
- tokio::spawn でチャット用タスクと要約/独り言キャッシュ生成タスクを分離
- キューで管理し、メイン応答の描画中はバックグラウンドタスクを待機（低優先度）

## 4. 開発ステップ
1. ディレクトリ構造と設定ファイル（ghost.json / shell.json / dic/main.yaml、schema_version 含む）の定義
2. Tauri Rust バックエンド: 透過/セーフモード/クリック透過、keyring、SQLite（履歴・独り言キャッシュ、非同期対応）
3. フロントエンド: 透明対応 UI、プリロード、思考中表示・吹き出し・チャットログ、TTS フックを意識した描画処理
4. 対話エンジン: 低負荷モード、LLM アダプター（JSON シングルコール・フォールバック）、コンテキスト移行、独り言キャッシュ

## 5. リリース向け追加機能（2026-06 採用・実装済み）

### 5.1 常駐基盤
- システムトレイ: 表示/非表示・モード切替・静音・セーフモード・設定・終了。左クリックで表示トグル
- ウインドウ位置の保存・復元。モニタ構成変更で画面外なら主モニタ中央へフォールバック
- OS ログイン時自動起動（設定でオン/オフ、tauri-plugin-autostart）
- 初回起動オンボーディング: first_boot イベント台詞で「API キー無しでも動く」「右クリックで設定」を案内
- ウインドウを閉じる操作は非表示に変換（終了はトレイから。常駐アプリとして振る舞う）

### 5.2 イベント駆動の存在感（伺か的中核）
- 辞書 schema v2 `events:`: first_boot / boot / quit / poke_main / poke_sub / poke_rapid /
  nade_main / nade_sub / idle。各台詞に when 条件（時間帯・日付）を付与可能（時間帯別挨拶・正月などの日付イベント）
- つつき反応: ダブルクリック=つつき、4 連打以上=連打反応（辞書駆動・LLM 不使用で即応答）
- 撫で反応: キャラ上をボタン無しで往復＝撫で。往復運動の指標（方向反転・累積移動量・局所性・継続時間）で
  「ただのカーソル通過」と区別し、つつき（びっくり）とは別の穏やかな反応を返す（辞書駆動・即応答）
- 放置反応: 30 分無操作で一言（操作でリセット）

### 5.3 ユーザー長期記憶
- user_profile テーブル（上限 50 件）。LLM 出力の remember 配列で自動抽出し、
  システムプロンプトへ常時注入。設定パネルで閲覧・手動追加・個別削除

### 5.4 配慮・運用
- 静音モード（トレイ/設定から切替）: 自発的発話を停止。全画面アプリ検知時の自動静音（既定オン）
- 会話履歴の全削除（記憶を含めるか選択可、2 段階確認）
- 更新通知: update_feed_url の JSON を起動時確認し、新バージョンをキャラの台詞で一度だけ通知

### 5.5 時事ネタ雑談・興味分野・オンボーディング（既定オフ・オプトイン）
- 興味分野（interest_topics・最大 20 件）に基づき、定期的に web（既定: Google ニュース検索 RSS）から見出しを取得
- 取得した見出しは高度モードの独り言生成プロンプトに折り込む（追加 API コールなし）。事故・災害・死・戦争・重い政治など
  ネガティブな見出しには触れない安全ガードをプロンプトに明記。低負荷モードは見出しを発話しない
- 興味分野はプロンプトにも注入し会話の話題選びに反映。プライバシー: 興味語が検索サービスへ送られる（既定オフで担保）
- 初回起動時にプロフィール聞き取り（フォーム型）: ニックネーム・興味分野・話し方・時事ネタ ON/OFF。設定から再実行可

### 5.6 音声入出力・ツール（いずれも既定オフ）
- 音声合成 TTS: **別アプリ/サーバの起動なしに ugga 単体で動く**ことを要件とする。既定エンジンは **埋め込み voicevox_core**
  （公式プリビルド C API を libloading で実行時ロード、CPU合成、GPU不要）。初回のみ公式ダウンローダ経由で資産
  （voicevox_core.dll / onnxruntime / Open JTalk辞書 / .vvm モデル）を `dirs::data_dir()/ugga/voicevox` 配下へ自動DL。
  全発話経路（応答・独り言・つつき・撫で・挨拶・時事ネタ・リマインダー）に声が乗る。**発話テキストは外部送信しない**。
  **VOICEVOX 音声モデル利用規約に基づき、声がVVMのときは「VOICEVOX:キャラ名」のクレジットを画面下端中央に常時表示する**。
  - 別途起動済みの外部 VOICEVOX エンジン(HTTP)に接続するオプションも残す（後方互換）
  - 上級向けに OpenAI互換 `/v1/audio/speech` 接続オプション（VoiceDesign キャプションを `instructions` で送信。
    参照音声クローンは未対応＝著作権リスク回避。外部サーバ前提なので「単体で動く」要件は満たさない）
  - メイン/サブ別話者・話速・音量・声のテスト。自動再生は WebView2 のブラウザ引数で許可、再生キューにウォッチドッグ
  - boot 時・設定変更時に合成器を事前 init（初回発話の数秒ラグを消す）
- 音声入力 STT（OpenAI 互換 /audio/transcriptions）: 実装済みだが**現在は設定 UI を非表示にして事実上無効化**
  （フロントの STT_UI_ENABLED フラグ。再有効化は true に戻すのみ。バックエンド実装は温存）
- ツール（実用アシスタント化）: 高度モード向け。現在日時の注入（時刻/曜日を即答）、リマインダー（「N 分後に教えて」→
  タイマーで発話・静音中も鳴る）、クリップボード補助（📋 ボタン押下時のみ読み取り、翻訳/要約/説明）。追加 LLM コールなし

### 5.7 対話・設定の UX 改善
- 設定画面のカテゴリ分け: 上部プルダウンで 6 カテゴリ（基本/会話・ふるまい/音声/見た目/記憶/システム）を切替。保存で自動クローズ
- 吹き出しの表示時間: セリフ総文字数に比例して延長。TTS 発話中は表示を保持。全ターンの描画・発話が終わってから
  単一タイマーで一括消去（個別消去しない）
- 掛け合いパターン（1.main→sub / 2.sub→main / 3.main→sub→main / 4.sub→main→sub）: 番号が小さいほど高確率。
  順番は内容の性質なので高度モードでは生成前に決めてその順で書かせる（パターン 2-4 は高度モードのみ）。
  辞書・独り言・リマインダーは固定文のため常に 1。3 ターン目は最初の話者の同じ吹き出しに 1 行空けて追記（吹き出しは 2 つのまま）

### 5.8 口パク（リップシンク。Tier 2）
- 静止画の閉口/開口 2 フレーム切替（連続アニメではない）。pose 画像 `<name>.png` の隣に `<name>_talk.png`
  があれば開口フレームとして自動使用（シェル定義変更不要・schema 据え置き・無いシェルは口パクせず従来どおり）
- TTS 有効時は再生中の音声振幅（AnalyserNode）に同期して今話しているキャラの口を開閉。
  TTS 無効時はタイプライター描画中に一定間隔で近似パクパク

### 5.9 ポモドーロ（集中タイマー。Tier 2）
- 集中(work)→休憩(break)をラウンド数だけ繰り返す（既定 25 分 / 5 分 / 4 回。設定で変更可）
- 集中中は自発発話（ランダムトーク・放置反応）を停止＝集中モード（should_stay_quiet に集中フラグを追加）。
  停止・全完了で自動的に通常へ復帰
- 節目（集中開始/集中終了→休憩/休憩終了→再開/全完了）でキャラが声かけ（辞書 events、ユーザー起点なので静音中も再生）
- 操作: 画面上部のカウントダウンバッジ（クリックで停止）/ 設定「会話・ふるまい」/ トレイの開始・停止
