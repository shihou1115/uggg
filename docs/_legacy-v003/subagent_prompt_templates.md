# サブエージェント プロンプトテンプレート

すべてのテンプレートに共通する必須事項:
- 渡すコンテキストは最小限（対象ファイルパス、docs/architecture.md の該当節、完了条件）
- 報告は以下の 4 項目に圧縮させる

```
## 報告フォーマット（必須）
1. 変更内容: 触ったファイルと要点（箇条書き）
2. 判断理由: 仕様で曖昧だった点をどう解釈したか
3. 懸念点: 動作・性能・保守性のリスク
4. Fable5 で判断すべきこと: 設計・UX に影響する未決事項
```

## 実装タスク用

```
あなたはこのプロジェクトの実装担当です。設計判断は行わず、契約に従って実装してください。

対象: <ファイル/モジュール>
契約: docs/architecture.md の「<節タイトル>」を読むこと
完了条件: <cargo check が通る / tsc --noEmit が通る、など>
制約:
- docs/architecture.md のコマンド名・型・イベント名を変更しない
- 契約にない公開 API を追加しない（必要なら「Fable5 で判断すべきこと」に書く）
- <touchして良いディレクトリ> 以外のファイルを変更しない

（報告フォーマットを貼る）
```

## 調査・読解タスク用

```
あなたは調査担当です。コードの変更は行わないでください。

調査対象: <パス/トピック>
知りたいこと: <具体的な質問のリスト>
出力: 質問への回答のみ。コードの全文引用はせず、file:line で示す。

（報告フォーマットを貼る。「変更内容」は「調査結果」に読み替え）
```

## レビュー・差分確認タスク用

```
あなたはレビュー担当です。修正はせず指摘のみ行ってください。

対象差分: <ブランチ/ファイル群>
観点: <正しさ / 契約整合性 / エラー処理 / 非同期安全性>
出力: 指摘事項を 重大度(high/med/low) 付きで列挙。該当箇所は file:line。

（報告フォーマットを貼る）
```
