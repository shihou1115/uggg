## 1. プロダクトコンセプト

### 1.1 ugg とは
「伺か」コンセプトを Tauri v2 で再構築したデスクトップマスコット。デスクトップに常駐する2体（または1体）のキャラクターが、自発的に話したりユーザーと対話したりする。

### 1.2 コアアイデンティティ
- **マスコットの存在感**: 起動挨拶・独り言・放置反応・つつき/撫で反応で「そこにいる」感を出す。
- **掛け合い対話**: メイン/サブ2体の会話表現が基本。
- **無料でも動く**: API キー無しでも辞書ベースの会話で楽しめる。
- **プライバシー優先**: 発話テキストは外部送信しない（クラウド LLM 経路を除く明示的同意済みの範囲のみ）。

### 1.3 ターゲット
- **OS**: Windows 10/11 x64 のみ
- **ハードウェア**: 通常のCPUのみで全機能動作（一部 GPU 拡張機能あり、§4.5.1 参照）

## 2. 採用済み技術選定（再調査しない）

| 領域 | 選定 | 理由 |
|---|---|---|
| プラットフォーム | Tauri v2 + Vanilla TypeScript + Rust | v0.0.3 実証済 |
| データストア | SQLite + keyring + ファイル資産 | 同上 |
| TTS（既定） | voicevox_core 埋め込み（libloading + プリビルドC API） | 無サーバ・CPU合成 |
| TTS（オプション） | Irodori-TTS サイドカー同梱（GPU必須） | キャプションVoiceDesignの上位品質 |
| クリック透過 | アルファマスク方式（8px グリッド + ポーリング） | 透明部分の誤クリック防止 |
| 対話モード | 二モード制（low=辞書 / advanced=LLM） | API キー無しでも動かす |
| 配布 | NSIS インストーラ（currentUser） | v0.0.3 実証済 |

## 3. 横断方針（全機能に適用）

### 3.1 ゴースト発話原則
**システム的なメッセージはできる限りキャラクター（ゴースト）に喋らせる**。トースト・ダイアログは補助・フォールバック扱い。

対象例:
- コスト警告（80%到達・上限超過）
- モード自動降格の通知
- 更新通知
- 自動DL完了・失敗
- 軽微なエラー（重大なものはトーストで二段表示）

辞書 events に対応キーを定義し、各台詞バリエーションを作者が書ける構造。発話できない状況（ゴースト未ロード・起動直後・致命的エラー）は従来のトースト/ダイアログにフォールバック。

### 3.2 「無サーバ」要件
**ユーザーが ugg を使うために、別アプリ・別サーバを手動起動する必要はない**。
- voicevox_core: アプリに埋め込み
- Irodori-TTS: サイドカーとして同梱（ユーザーから見れば「ugg の一部」）
- LLM API: 例外（クラウド/ローカルとも、ユーザーが API キー or LLM サーバを別途用意）。これはユーザーの選択で動かす対象なので「無サーバ」の対象外と整理。

### 3.3 プライバシー
- 発話テキスト・合成音声は外部送信しない。
- LLM プロバイダへの送信のみクラウドへ出る（ユーザー同意済み・既定ローカル/オフライン優先）。
- 時事ネタの検索キーワードは検索サービスへ送信（既定オフ・オンボーディング同意必須）。
- データのエクスポート・削除をユーザーが自由にできる。

## 4. 機能仕様

### 4.1 キャラクター・ウインドウ

#### 4.1.1 キャラ構成（2体だがサブは任意）★
- メインキャラとサブキャラの 2 体構成を基本とする。
- 配置は伺か風: 下端揃えでタスクバーの上に立つ。**既定配置はメインが画面右端・サブがその左隣**（デフォルトシェルは main 256×384 / sub 96×144 でメイン大・サブ小）。
- **キャラはそれぞれ独立してドラッグで X 軸方向にのみ移動できる**（§4.3.4）。Y は作業領域下端（タスクバー上端）で固定。移動範囲は同一モニタ内。
- **シェル/ゴーストはサブを省略可能**。サブ無しゴーストでは:
  - 画面にメインのみ表示
  - 掛け合いパターンはパターン1（main単独）相当に縮退
  - 撫で/つつきは main のみ
  - 設定 UI の「サブの声」等は無効化または非表示
- データスキーマ（shell.json）と辞書スキーマは「サブが存在しないゴースト」を表現可能にする（Phase 2）。

#### 4.1.2 キャラ画像表示（全 pose プリロード）
- 全 pose の `<img>` を data URL でプリロードし、`visible` クラス切替で表現。
- 切替時のチラつきはほぼゼロを目標。

#### 4.1.3 吹き出し描画 ★
- **吹き出しはキャラクターの左横に表示**(伺か風)。上端をキャラの顔の高さ（キャラ上端 + 高さの 12%）に合わせ、しっぽはキャラ側を向く。main/sub 両表示で重なる場合は main を上・sub を下へ退避。
- キャラが画面左端付近で左横に収まらない場合は**右横に反転表示**（しっぽも反転）。
- **タイプライター描画**（1文字ずつ表示）。
- **描画速度はユーザー設定で可変**（速い / 普通 / 遅い / 一気に表示）。
- main → sub の順に逐次表示、重ならない。
- **掛け合いパターン3/4 では 3 つ目の吹き出しを独立表示**（v0.0.3 の「同じ吹き出しに追記」方式は廃止）。
- 新発話で interrupt（古い音声・描画を打ち切る）。
- 表示継続時間はセリフ総文字数に比例。
- 全ターンの描画+発話完了後に一括消去。

#### 4.1.4 口パク（振幅駆動のみ）★
- TTS 有効時のみ動作。再生振幅（AnalyserNode RMS）でリアルタイム駆動。
- **TTS 無効時の口パクは無し**（v0.0.3 のタイマー近似を廃止）。
- pose 画像の隣に `<name>_talk.png` があれば開口フレームとして使用。無いシェルは口パクなし。

#### 4.1.5 表示スケール ★
- 設定 UI で 0.5〜2.0 の範囲で可変。
- スケールはキャラ画像のみに適用（ウインドウはステージ方式 §4.1.6 で全幅固定のため連動しない）。拡縮の基準点はキャラの足元左（bottom left）。
- **要件: 吹き出しのテキスト・フォントサイズはスケールの影響を受けず、視認性を維持する**（v0.0.3 で課題化された問題）。
- 実装方針（Phase 2 で設計）: キャラ画像と吹き出しのレイヤーを分けて scale を別管理、または逆補正。

#### 4.1.6 ステージ（ウインドウ）とキャラ位置の永続化
- ウインドウは**モニタ作業領域の全幅 × 高さ 600 の透明ステージ**として**作業領域下端に固定**する（＝キャラの足元が常にタスクバー上端に乗る）。ユーザーがウインドウ自体を動かす手段は無い。
- ステージを置くモニタは前回終了時のものを記憶（`window_pos` キー）。無ければ・消滅していれば主モニタ。
- モニタ構成・解像度・タスクバー高さの変更は 1 秒間隔の監視で検知し、自動で再ドックする。
- **キャラごとの X 位置**（ステージ内 CSS px、視覚ボックス左端）を `char_pos` キーに永続化する。保存はドラッグ終了時（mouseup）に即時。起動時に復元し、ステージ幅に収まるよう clamp。保存が無いキャラは既定配置（§4.1.1）。

#### 4.1.7 タスクトレイ ★
- アイコン常駐、左クリックでウインドウ表示/非表示。
- 右クリックでメニュー（**セーフモード項目は除く**）:
  - 表示/非表示
  - モード切替（low / advanced）
  - 静音モード
  - 設定を開く
  - 終了
- メニューのチェック状態は設定変更時に同期。

#### 4.1.8 セーフモード（廃止）★
- v0.0.3 のセーフモード（不透明・枠ありで起動する保険機能）は廃止。
- ウインドウ生成は単一ラベル・単純構造に。

### 4.2 対話

#### 4.2.1 二モード制
- **low モード**: 辞書のみ。API キー不要・無料・オフライン。
- **advanced モード**: LLM 経由で動的生成。
- ユーザーは設定で手動切替可。
- 自動降格条件: API エラー連続 / コスト上限超過 → 一時 low に降格 → 一定時間後に復帰試行。

#### 4.2.2 LLM プロバイダ（OpenAI 互換のみ）★
- **OpenAI 互換 API のみサポート**。
- 設定: モデル名 / `base_url`（任意） / API キー（keyring）。
- これにより以下が同じ仕組みで接続可:
  - OpenAI 公式
  - Grok（OpenAI 互換）
  - LM Studio / Ollama（ローカル LLM、OpenAI 互換エンドポイント）
  - その他 OpenAI 互換サーバ
- **Anthropic SDK 専用ルートは廃止**。

#### 4.2.3 辞書スキーマ v3 ★
- v0.0.3 の schema v2 を整理して v3 を新規策定（**v2 からの自動マイグレーションは設けない**、作者に書き直しを求める）。
- 整理方針:
  - セクションを統合・最小化（recall_talk / update_talk 等の使い分けを見直し）
  - events キーの乱立を抑制（推奨命名・部位粒度を C-2 に合わせて簡素化）
  - 横断方針 §3.1 に対応する system 系 events を整理・追加
- スキーマ詳細は Phase 2 で策定。

#### 4.2.4 掛け合いパターン ★
- パターン1: main → sub
- パターン2: sub → main
- パターン3: main → sub → main（**3つ目の吹き出し**で表示）
- パターン4: sub → main → sub（**3つ目の吹き出し**で表示）
- **新パターン: 問いかけパターン**（極低確率発生、ユーザー応答を期待）
  - 例: 「最近どんなことに興味あるの？」「コーヒー派？紅茶派？」
  - ユーザー応答時の処理（user_profile への自動反映、follow-up 発話）は Phase 2 で設計。
  - 応答が来なかった場合のタイムアウト挙動も Phase 2 で設計。
- サブ無しゴーストではパターン2/3/4 は使えない（自動的にパターン1へ縮退）。
- advanced のみ可変、辞書系は常にパターン1。
- 番号が小さいほど高確率（重み付け）。

#### 4.2.5 長期記憶（user_profile）★
- ユーザーの情報を保存。LLM 呼び出しの system prompt に注入。
- 設定パネルで閲覧・追加・削除。
- オンボーディングで nickname / 興味 / 話し方 / 時事ネタ ON/OFF を聞き取り → 自動投入。
- **新機能: 会話からの自動抽出**（advanced のみ）
  - LLM 呼び出し時に「記憶すべきサマリ」を副産物として抽出
  - user_profile に自動保存
  - 抽出ロジック・トリガー条件は Phase 2 で設計
- **容量管理**: 件数/文字数の上限・古いものの自動破棄/要約を Phase 2 で設計。

#### 4.2.6 会話の自動要約（記憶に統合）★
- v0.0.3 の `context_summaries` テーブルは廃止。
- 過去会話の要約は §4.2.5 の user_profile 自動抽出に統合する。
- recall_talk のトリガーも user_profile から派生。

#### 4.2.7 コスト管理 ★
- LLM 呼び出しのコストを `api_usage` テーブルに記録。
- 月額上限 `monthly_limit_usd` を設定。
- **80% 到達時に警告**:
  - トースト + **ゴースト発話**（横断方針 §3.1）
  - 辞書 events に対応キーを追加（例: `cost_warning_80`）
- **上限超過時に自動降格**:
  - low モードへ降格
  - **ゴースト発話**で通知（黙って降格しない）
- 次月リセットで復帰。
- ローカル LLM 利用時は記録のみ（料金は 0）。

#### 4.2.8 API キー管理
- keyring に保存（`service = "ugg"`, `user = provider 名` 等）。
- 設定パネルで登録（伏せ字表示）、「保存済み」状態を表示。
- 1 プロバイダ 1 キー。

### 4.3 ユーザー操作

#### 4.3.1 クリック種別の判別と入力導線 ★
- 最後のクリックから 250ms 待って確定:
  - 1 回 → **入力導線**（下記）
  - 2-3 回 → つつき反応
  - 4 回以上 → 連打反応
- mousedown 後の移動 4px 超でドラッグ扱い（クリック判別を抜けてキャラ移動 §4.3.4 へ）。
- **入力導線**: クリックされたキャラが「なにか用かな？」のような**入力促しの反応**（辞書 `input_prompt`、そのキャラ単独の短い発話。TTS 対象・chat_log 記録あり）を返し、**チャット入力欄をそのキャラのバルーンの上側に表示**する。
  - 促しの吹き出しは通常の表示時間で消さず、**入力欄が閉じるまで保持**する。
  - 入力欄が開いている状態での 1 クリック・Esc は入力欄と促し吹き出しを閉じる。送信時・新しい発話でも促しは置き換わる。
  - キャラ以外（吹き出し等）のクリックは main の導線として扱う。辞書に `input_prompt` が無いゴーストでは促し無しで入力欄だけ開く。
  - 入力欄の配置はバルーンと同じ幾何に従う（キャラ左横基準、画面左端付近では右横へ反転、キャラのドラッグに追従）。

#### 4.3.2 つつき部位判定（縦のみ）★
- 縦 3 区分: head / chest / body（**横判定は廃止**）。
- shell.json で `poke_regions.head_max` / `chest_max` を指定（横の `left_max` / `right_min` は廃止）。
- 辞書 events 探索順: `poke_<t>_<v>` → `poke_<t>`。

#### 4.3.3 撫で（ホバー往復判定）
- ボタン無しでキャラ上を往復するホバージェスチャを検知。
- 判定指標の合わせ技: 方向反転回数 ≥2 / 累積移動量 / 局所性 / 最低継続時間 / cooldown。
- 部位判定は §4.3.2 と同じ（縦のみ）。
- **パネル (設定・読み上げ・チャットログ・確認・オンボーディング) 表示中は無効**。
  パネル操作でカーソルが背後のキャラ上を往復して誤発話するため。

#### 4.3.4 ドラッグでキャラ移動（キャラ単位・X 軸のみ）★
- mousedown したキャラ（部位判定と同じヒットテスト）だけを動かす。**main / sub は独立して移動**。
- 移動は **X 軸方向のみ**。Y はステージ下端（タスクバー上端）に固定（伺かと同仕様）。
- mousedown 後 4px 超でドラッグ開始。移動範囲はステージ（モニタ作業領域）内に clamp。
- ドラッグ終了（mouseup）で位置を即時保存（§4.1.6）。
- 吹き出し表示中はキャラに追従する。
- キャラ以外（吹き出し・透明部）のドラッグでは何も動かさない。`data-tauri-drag-region` は使わない。

#### 4.3.5 右クリック → バルーンメニュー ★
- 独立したコンテキストメニュー UI は廃止し、**メニューはメインキャラのバルーン内に表示**する（伺か風）。
- **メインを右クリック**: メインが前口上（辞書 `menu_prompt.main`、例「なにか用かな？」。TTS 対象・chat_log 記録）を発話し、**続けて同じバルーン内に**メニュー項目を表示する。
- **サブを右クリック**: サブが誘導セリフ（辞書 `menu_prompt.sub`、例「用事ならメインに頼んでくれ」）を自分の吹き出しで発話した後、メイン右クリックと同じ挙動に遷移する（サブの吹き出しは表示したまま）。
- 項目（実装準拠）: モード切替（low ⇔ advanced）/ 静音モード切替 / ポモドーロ開始・停止 / テキスト読み上げ / 設定を開く / ウインドウを隠す / 終了。ゴースト/シェル切替は設定パネルから行う。
- 閉じる: 項目実行・Esc・メニュー外クリック。新しい発話（自発発話等）でも置き換わる。メニュー表示中の左クリックは「閉じる」専用（入力導線 §4.3.1 やつつきには流さない）。
- キャラ以外の右クリックは何もしない（WebView 既定メニューの抑止のみ）。
- 辞書に `menu_prompt` が無いゴーストでは、セリフ無しでメニューのみ表示（サブ右クリックは直接メインのメニューへ）。

### 4.4 存在感系イベント

#### 4.4.1 起動挨拶
- 初回 = `events.first_boot`（オンボーディング兼）。
- 2 回目以降 = `events.boot`（時間帯条件 / 日付条件で分岐）。
- 静音モードでも起動挨拶は鳴らす。

#### 4.4.2 終了挨拶
- トレイ「終了」時に `events.quit` を再生してから exit。
- 強制終了では当然鳴らない。

#### 4.4.3 放置反応
- 60 秒間隔チェック、最終操作から 30 分で 1 回 idle 発火。
- 静音中・busy 中は持ち越し。
- ユーザー操作でリセット。

#### 4.4.4 ランダムトーク（独り言）★
- 一定間隔でキャラが独り言を発話。
- **既定間隔: 10 分**（v0.0.3 の既定 0=無効から変更）。
- 0 設定で無効化可能。
- advanced モードでは LLM 生成 + キャッシュ補充。
- low モードでは辞書 `random_talk` から選択。
- 静音条件下では発火しない。

#### 4.4.5 ポモドーロ ★
- 集中(work) → 休憩(break) をラウンド数だけ繰り返す。
- 既定 25 分 / 5 分 / 4 回（設定変更可）。
- 集中中は静音扱い（ランダムトーク・放置反応を止める）。
- 辞書 events: `focus_start` / `focus_end` / `break_end` / `pomodoro_done`。
- **focus_end / pomodoro_done では「ねぎらいの台詞」を充実**させる（辞書側で台詞バリエーション拡充）。
- バッジ表示（画面上部中央、`.solid`）。
- トレイ・右クリックメニューから start/stop。

#### 4.4.6 時事ネタ雑談
- RSS（既定 Google ニュース検索）から興味分野の見出しを取得。
- **advanced モードの独り言生成のついで**に織り込む（追加 LLM コストなし）。
- 暗い見出し（事故・災害等）をフィルタ。
- low モードでは時事ネタ無し。
- 取得間隔は 30 分以上にクランプ。
- 興味分野は `interest_topics` テーブル（user_profile との統合は Phase 2 検討）。
- **既定オフ**・オンボーディングで明示同意必須。

#### 4.4.7 更新通知
- `update_feed_url` の JSON を起動時に取得。
- 現バージョンより新しければ辞書 `update_talk` で**一度だけ**ゴーストが通知。
- 既定オフ（フィード URL 未設定なら無効）。

#### 4.4.8 静音モード
- `quiet_mode` 設定 ON で自発的な発話を停止。
- ユーザー操作への応答は通常通り。
- 起動挨拶・終了挨拶は静音下でも鳴る。
- トレイ・右クリックメニューから切替。

#### 4.4.9 フルスクリーンアプリ自動静音 ★
- `auto_quiet_fullscreen` 設定 ON で、フォアグラウンドウインドウがモニタ全面なら自動静音扱い。
- ゲーム・プレゼン・動画視聴中は黙る。
- Win32 API で検知、デスクトップシェル（Progman/WorkerW）は除外。
- **既定 OFF**（v0.0.3 の既定 ON から変更、ユーザーが明示的に ON）。

### 4.5 音声・補助

#### 4.5.1 TTS ★
- **2 エンジン体制**:
  - **voicevox_core**（既定、必須）: 公式プリビルド C API を libloading で実行時ロード。CPU合成・無サーバ。
  - **Irodori-TTS**（オプション、上級）: サイドカー同梱、GPU必須、不安定許容。
- **共通動作**:
  - 「ユーザーが別途サーバー/アプリを起動する必要はない」を両方とも満たす。
  - 失敗時は声なしで会話継続（アプリは落ちない）。
  - 全発話経路（応答・独り言・つつき・撫で・挨拶・時事ネタ・リマインダー）に乗る。
- **voicevox_core 詳細**:
  - 資産（dll / onnxruntime / Open JTalk 辞書 / .vvm）は **初回 DL** で取得
  - 規約同意必須、GitHub PAT で API レート制限緩和（任意）
  - クレジット表示「VOICEVOX:話者名」を画面下端中央に常時表示（VVM 規約準拠）
  - 事前 init（boot 時・設定変更時）で初回発話のラグを解消
- **Irodori-TTS 詳細**:
  - Python + PyTorch + CUDA + モデルをサイドカー同梱方式で配布
  - **初回 DL 方式**（軽量インストーラ → 設定で有効化時に 5〜10GB DL）
  - GPU 検出失敗時は voicevox_core にフォールバック + ユーザーに通知
  - **参照音声クローン方式**（声質の一貫性を確保）
    - 参照音声がある場合: 保存済み参照音声でクローン合成
    - 参照音声が無い場合: キャプション（VoiceDesign）から参照音声を生成 → 保存 → 以降はクローン合成へ切替
    - 参照音声の供給源は**自前生成のみ**（外部音声の取り込みは行わない＝著作権リスク回避）
    - 参照音声の再生成（キャプション変更で作り直し）・削除 UX は Phase 2 で設計
  - **漢字→ひらがな前処理**: Irodori-TTS は漢字の発話精度が不安定なため、合成前にテキストをひらがなに変換する前処理を入れる
    - 変換手段の候補: voicevox_core が同梱する Open JTalk 辞書の流用 / 形態素解析ベース（Phase 2 で選定）
    - 変換漏れ・誤読は許容（完全な読み付与は目指さない）
    - voicevox_core エンジン側ではこの前処理は不要（内部で読み解析を行うため）
  - **絵文字アノテーションの保護**: Irodori-TTS V3 は入力テキスト中の特定絵文字（45 種、upstream `EMOJI_ANNOTATIONS.md` 準拠）で感情・スタイル・効果音を制御できる。前処理はこれらの絵文字を除去せず、位置を保ったまま合成テキストに残す
    - 対応絵文字は upstream の一覧をホワイトリストとして保持（対応外の絵文字は従来通り前処理で落ちる）
    - 同一絵文字の連続（強調）も保持する
    - voicevox_core エンジン側では絵文字は解釈されないため保護しない（従来通り読み飛ばし）
  - **詳細実装は Phase 2 で**: Python ランタイム配布方式 / モデル配布 / サイドカープロセス管理 / GPU 検出 / 参照音声管理（生成・保存・再生成・削除） / **漢字→ひらがな前処理の実装方式** / 設定UI
- 設定:
  - エンジン選択（VOICEVOX / Irodori-TTS）
  - 話者選択（メイン / サブ）
  - 話速・音量
  - Irodori-TTS のキャプション入力（参照音声の初回生成・再生成に使用）
  - Irodori-TTS の参照音声管理（生成・再生成・削除）
  - 規約同意・GitHub PAT（voicevox_core 用）
- 共通の再生制御:
  - 逐次キュー（main → sub が重ならない）
  - 新発話で interrupt
  - WebView2 自動再生ポリシー対策（`--autoplay-policy=no-user-gesture-required` + ウォッチドッグ）

#### 4.5.2 STT（廃止）★
- v0.0.3 で実装したが MVP では搭載しない。コード・設定・コマンドを全て削除する。
- 将来必要になれば §6 将来課題から検討。

#### 4.5.3 ツール（時刻・リマインダー・クリップボード）
- `tools_enabled` トグルで 3 つまとめてオンオフ（個別切替なし）。
- advanced モード前提。
- **時刻注入**: 「今何時?」「今日何曜日?」に LLM が答えるための system prompt 補強。
- **リマインダー**: 「N 分後に教えて」→ タイマー登録 → 時間で発話（**静音中も鳴る**）。
- **クリップボード補助**: 入力欄の📋ボタン押下時のみ読み取り、「翻訳して」「要約して」「説明して」等に応える。**📋 ボタンは `tools_enabled = false` のとき非表示**。
- 既定オフ。

#### 4.5.4 自動起動
- OS ログイン時の自動起動（`tauri-plugin-autostart`）。
- 既定 OFF、ユーザーが設定で ON。

#### 4.5.5 データエクスポート / 履歴クリア
- **エクスポート**: 会話ログ・記憶（任意）・コスト履歴を JSON でダウンロードフォルダに書き出し。
- **履歴クリア**: 会話ログ・独り言キャッシュを削除。記憶を含めるかはオプション。
- 設定パネルから手動実行。

#### 4.5.6 ゴースト・シェル切替 ★
- 設定パネルで複数のゴースト・シェルから選択切替。
- **ホットリロード（ファイル編集の即時反映）は廃止**。作者は再起動して確認する。
- **検討: ドラッグ&ドロップ展開**
  - ゴースト/シェルの zip ファイルやフォルダをアプリにドロップ
  - 自動的に `ghosts/<id>/` or `shells/<id>/` に展開
  - 「再起動して使えるようにする」を明示提示
  - 実装可否・対応形式は Phase 2 で決定（zip 中身検証・上書き確認 UX 含む）

#### 4.5.7 チャットログパネル
- 簡易ログパネル（開閉式、`.solid`）。
- `get_chat_log(limit)` で新しい順 N 件取得。
- 設定パネルから開閉。

#### 4.5.8 テキスト読み上げツール（詳細: docs/text-reader-spec.md、台本形式: docs/script-reader-spec.md）
- 右クリックメニュー「テキスト読み上げ」で専用パネルを開く。パネル**表示中のみ**、
  `.txt` ファイルの DnD を受理してメインキャラの声で読み上げる（複数 DnD は先頭 1 件）。
- 入力: `.txt` のみ、1MB 上限、UTF-8 (BOM 有無) / Shift_JIS 自動判定。
- 長文対策: 文境界優先・最大 120 トークンのチャンク分割（`reader_load_text` コマンド）。
  Irodori 絵文字アノテーションはどの分割経路でも分断しない。
- 合成はチャンクごとに既存 `synthesize_voice`（= 前処理・絵文字保護・フォールバックが適用）。
  再生は先読み 1 のパイプライン、失敗チャンクはスキップして件数表示。
- 読み上げ中は自発発話を抑制（`set_reading_active` コマンド → 静音判定に追加）。
  リマインダーの静音特例は従来通り。停止ボタン / パネルクローズで即停止。
- 追加コマンド: `reader_load_text(path) -> Vec<ReadingChunk>` / `set_reading_active(active)`。
  `synthesize_voice` に `caption: Option<String>` を追加（Irodori 実モデル経路のみ使用）。
- **台本形式対応**: `.md` 台本（`speakers` / `lines` ブロック必須 + `defaults` 任意）も
  受理する。話者 ID を ugg の slot（main / sub）へマッピングし、行ごとに話者を切り替える。
  行ごとの `speed`（再生側 playbackRate に合成）、`caption`（声質・演技指示。Irodori 実モデル
  時のみ有効、無効環境ではパネルに注記を表示して読み上げ）、`pause_after`（行間の間）を制御。
  不正な台本はロード時に全行検証して fail-fast（部分再生しない）。詳細は docs/script-reader-spec.md 参照。

## 5. 配布

- **形式**: NSIS インストーラ（`currentUser` モード、日本語ロケール）。
- **初回起動時 DL** が必要なもの:
  - voicevox_core 資産（c_api / onnxruntime / dict / models 合計数百MB）
  - Irodori-TTS（オプション、Python+PyTorch+CUDA+モデル 数GB）
- **インストーラサイズ目標**: 5〜10 MB（資産は全部初回DL）
- **インストール先**:
  - 実行ファイル: `%LOCALAPPDATA%\ugg\`
  - データ: `%APPDATA%\ugg\`（DB、TTS 資産、ログ）
  - keyring: OS 標準（Windows Credential Manager）
- **更新**: 自動更新は無し、`update_talk` 経由のゴースト案内のみ（手動 DL & 再インストール）。
