## 概要

v0.1.4 (2026-07-11) に続くメジャー機能リリース。**「日常支援 Tier S」全 4 機能**
（spec §4.6、設計正本 docs/daily-support-design.md v2）を追加した。ugg を
「生活状況を軽く把握し、必要なときだけキャラクターとして声をかける常駐アシスタント」
へ拡張する第一弾。基盤はすべて low（ローカル・決定論）で動作し、AI 非依存
（spec §4.2.1 不変条件）。すべての自発的通知は新設の**発話ガバナンス**に従う。

DB スキーマは v5 → v8（reminders 拡張 + reminder_log / todos / calendar_cache 新設。
migration は起動時自動、既存データは引き継がれる）。

## 変更点

### 統合リマインダー (M7)

- チャット自然文での登録を大幅拡張: 従来の「N 分後」に加え、**絶対時刻**（18時に洗濯物）・
  **相対日**（明日の朝ゴミ出し）・**日付**（7月20日の10時に病院）・**繰り返し**
  （毎日9時に薬 / 毎朝 / 毎週月曜9時 / 月・水・金の9時）に対応。時間帯マッピングは
  朝8:00 / 昼12:00 / 夕方17:00 / 夜20:00。平叙文の誤登録を防ぐ雑談語尾ガード付き
- **発火 ≠ 完了**: 発火後も「未完了」として残り、パネルで完了・無視・スヌーズ（10 分）を選べる。
  通知履歴も参照可能。繰り返しは発火後に次回へ自動スケジュール
- **起動時回収**: アプリ停止中に過ぎた期限を起動後にまとめて 1 発話で通知（「◯◯（ほか N 件）」）
- `tools_enabled`・advanced から独立した常時ローカル動作へ移行（spec §4.5.3 の予告どおり）。
  静音中も鳴る特例は維持

### ToDo・日課管理 (M8)

- **今日 / 今週 / いつか**の 3 区分、優先度 2 段階（★）のシンプルな ToDo。
  右クリック → 「予定・ToDo」パネルで操作
- **日課（↻）**: 完了しても翌日（毎日）・翌週（毎週）に自動復活
- キャラクター関与: 朝の件数告知（1 日 1 回）・完了時の労い・終了時に未完了があれば
  ひとこと（todo_quit）

### 状況対応型自発発話 + 発話ガバナンス (M9)

- **状況発話 4 カテゴリ**（すべて既定 OFF のオプトイン）: 休憩促し（連続 90 分ごと）/
  深夜利用の声かけ（23-5 時・1 晩 1 回）/ バッテリー低下（15% 以下・1 回）/
  ToDo の思い出し・滞留整理（午後・夕方）。検知はローカル完結
  （ウインドウ内容・キー入力・スクリーンは読まない）
- **発話ガバナンス**: すべての自発発話を単一ゲートで管理
  （ハード静音 → **夜間静音（新設・時間帯指定）** → カテゴリ OFF → 最低間隔 → 連投回避）。
  登録した通知（リマインダー・カレンダー）は「必ず届く」Notice として静音を越える
- **🔕 フィードバック**: 声かけ中の吹き出しの 🔕 で「いまのは邪魔」を伝えると、
  その種類の頻度が下がり 3 回で自動 OFF（設定で再 ON すると頻度もリセット）

### カレンダー参照 (M10)

- **ICS（ファイル / URL）の読み取り専用**取り込み。書き込み・OAuth なし、既定オフ、
  外部送信は ICS 取得リクエストのみ（spec §3.3）
- **開始前通知**（既定 15 分前、変更可）と終日予定の通知（当日朝 8 時）。静音中も鳴る
- 繰り返し予定は直近 7 日分を展開（毎日・毎週は完全対応、複雑な繰り返しは当日分のみ
  「繰り返し(一部)」表示）。今日・明日の予定を「予定・ToDo」パネルに表示
- 30 分ごと自動更新。オフライン時・取得エラー時は最後に取得した内容を保持

### UI・設定

- 右クリックメニューに「**予定・ToDo**」を追加（リマインダー / ToDo / 今日明日の予定の統合パネル）
- 設定に「**日常支援**」ページを新設（マスタスイッチ・リマインダー通知・状況発話 4 種と
  最低間隔・カレンダー・夜間静音）。「ツール」からリマインダー関連を移設
- 通知のトースト表示（ゴースト発話が使えない場合のフォールバック）
- 取扱説明書（同梱 manual.md）を v0.2 の機能に合わせて更新

## 既知の制限 (このリリースでは対応しない)

v0.1.4 の「既知の制限」をそのまま引き継ぐ。加えて本リリース由来:

1. **リマインダーの自然文解釈は決定論パーサ**であり、形態素解析は持たない。雑談が
   まれに誤登録されることがある（確認発話が出るので、パネルから削除すればよい）。
   曖昧表現の解釈は advanced 上乗せの将来課題
2. **カレンダーの複雑な繰り返し**（RRULE の一部）は当日分のみ「繰り返し(一部)」として
   表示する。任意タイムゾーン（VTIMEZONE）の完全解決は将来課題（現状は Z=UTC /
   それ以外=ローカル扱い）
3. **カレンダーのソース編集**（追加・削除）時はキャッシュを作り直すため、既知の予定の
   「通知済み」状態が一度リセットされる
4. 連続利用時間の計測は 60 秒ポーリングのため、休憩促しのタイミングが数分ぶれることがある
5. 状況発話の時間帯依存機能（深夜声かけ・ToDo 思い出し等）の長期実機観察は今後の
   運用で継続確認する（ロジックはユニットテストで担保済み）

## 動作要件

v0.1.4 と同一。

- Windows 10/11 x64
- 一般機能: 任意の x86_64 CPU、3 GB 以上の空き RAM
- voicevox_core (推奨): 数百 MB のディスク + 一般的な CPU
- Irodori-TTS (任意): NVIDIA GPU (CUDA 12.8 対応、VRAM 4 GB 以上)、10 GB 以上のディスク

## アップグレード手順

v0.1.x からのアップグレードは、新しいインストーラを上書き実行するだけでよい
(currentUser モード)。`%APPDATA%\ugg\` の既存データは引き継がれ、DB は初回起動時に
自動でマイグレーションされる (v5 → v8、非破壊)。新機能のうち状況発話・カレンダーは
既定 OFF なので、挙動が勝手に増えることはない（自然文リマインダーの拡張と朝の ToDo
告知のみ既定 ON）。

v0.0.3 プロトタイプからの DB マイグレーションは提供しない。
