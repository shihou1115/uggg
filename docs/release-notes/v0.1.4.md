## 概要

v0.1.3 (2026-07-09) に続くマイナーリリース。**「伺か」風の UI/UX への刷新**が中心。キャラクターの配置・サイズ・吹き出し位置を本家に寄せ、キャラを 1 体ずつ動かせるようにし、クリック/右クリックの導線をキャラの発話を伴うものに作り替えた。DB スキーマに破壊的変更はなく、既存の会話履歴・設定・TTS 資産はそのまま引き継がれる。Tauri コマンドは 3 件追加 (既存契約の変更・削除はなし)、辞書 v3 に 2 セクションを後方互換で追加した。

## 変更点

### 伺か風レイアウトへの刷新

- キャラクターを**画面右下（作業領域の下端＝タスクバー上端）にドック**し、既定でメインが右端・サブがその左隣に立つ配置にした。デフォルトシェルのサイズをメイン大・サブ小 (256×384 / 96×144) に調整。
- **吹き出しをキャラクターの左横（顔の高さ）に表示**し、しっぽがキャラを指すようにした。画面左端付近では吹き出しを右横へ自動反転する。
- ウインドウを「モニタ作業領域の全幅 × 高さ 600 の透明ステージ」として下端に固定する方式に変更。モニタ構成・解像度・タスクバー高さの変更には自動で追従する。

### キャラクターの個別移動（X 軸のみ・位置永続化）

- **メインとサブをそれぞれ独立してドラッグで左右に移動**できるようにした。高さはタスクバー上端で固定（伺かと同仕様）。移動範囲はモニタ内に収める。
- 位置はキャラごとに保存し、次回起動時に復元する（ウインドウ全体をドラッグする旧方式は廃止）。

### クリック/右クリックの導線刷新

- **キャラを 1 クリック**すると、そのキャラが「なにか用かな？」のような入力促しの反応を返し、**チャット入力欄をそのキャラの吹き出しの上側に表示**する。
- **右クリックを「吹き出し内メニュー」方式に変更**。メインを右クリックするとメインが前口上を発話し、続けて同じ吹き出し内にメニュー項目（モード切替・静音・ポモドーロ・読み上げ・設定・隠す・終了）が並ぶ。サブを右クリックした場合はサブが「用事はメインに」と案内した後、メインのメニューに遷移する。独立したコンテキストメニュー UI は廃止。
- コンテキストメニューの「このキャラと話す」項目を削除（1 クリックの導線に統合）。チャット入力欄の 📋 クリップボード貼り付けボタンは、設定の「ツールを有効化」が OFF のとき非表示にした。

## 契約の変更（後方互換）

- **Tauri コマンドを 3 件追加**: `set_char_positions`（キャラ X 位置の保存）、`input_prompt`（クリック時の促し発話）、`menu_prompt`（右クリックメニューの前口上/誘導）。いずれも新規追加で、既存コマンドの引数・戻り値は不変。
- **boot payload に `char_positions` を追加**（保存済みキャラ位置。未保存は null）。
- **辞書スキーマ v3 に `input_prompt` / `menu_prompt` セクションを追加**。どちらも省略可能で、無いゴースト（旧辞書含む）は促し無しで従来どおり動作する（`schema_version` は 3 のまま）。
- `window_pos` 設定キーの意味を「ウインドウ座標」から「ステージをドックするモニタの記憶」に変更。旧データがあっても移行処理は不要（そのモニタにドックするだけ）。

## 配布物の変更

- `bundle.resources` に変更なし（v0.1.3 と同一構成: ghosts / shells / python/sidecar.py / manual.md）。

## 既知の制限 (このリリースでは対応しない)

v0.1.3 の「既知の制限」をそのまま引き継ぐ。加えて本リリース由来:

1. **キャラのモニタ間移動は不可**: ステージは前回終了時のモニタ（初回は主モニタ）に固定され、キャラの左右移動は同一モニタ内に限る。別モニタへの移動 UI は未提供。
2. **右クリックメニュー表示中にポモドーロバッジ等の要素をクリックすると、メニューが閉じると同時に入力欄が開くことがある**（限定的な角ケース。次版で修正予定）。
3. 台本形式・Irodori-TTS β 扱い等、v0.1.3 / v0.1.1 / v0.1.0 の既知の制限はそのまま有効。

## 動作要件

v0.1.3 と同一。

- Windows 10/11 x64
- 一般機能: 任意の x86_64 CPU、3 GB 以上の空き RAM
- voicevox_core (推奨): 数百 MB のディスク + 一般的な CPU
- Irodori-TTS (任意): NVIDIA GPU (CUDA 12.8 対応、VRAM 4 GB 以上)、10 GB 以上のディスク

## アップグレード手順

v0.1.3 以前からのアップグレードは、新しいインストーラを上書き実行するだけでよい (currentUser モード、DB スキーマ変更なし)。`%APPDATA%\ugg\` の既存データはそのまま引き継がれる。キャラ位置 (`char_pos`) は初回のみ既定配置（メイン右端・サブその左）で表示され、以後ドラッグで調整できる。

v0.0.3 プロトタイプからの DB マイグレーションは提供しない。
