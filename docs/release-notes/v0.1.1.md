## 概要

v0.1.0 (2026-07-02) に続くマイナーリリース。**テキスト読み上げツールの追加**と、実モデル (Irodori-TTS) 初回発話時のクラッシュ修正が中心。機能追加は 1 件で、残りは安定性向上とドキュメント整備。DB スキーマ・Tauri コマンド契約に破壊的変更はない。

## 変更点

### 新機能: テキスト読み上げツール (spec §4.5.8 / docs/text-reader-spec.md)

- コンテキストメニュー「テキスト読み上げ」で専用パネルを開き、**パネル表示中のみ** `.txt` ファイルの DnD を受理してメインキャラの声で朗読する (複数 DnD は先頭 1 件)。
- 入力: `.txt` のみ、1MB 上限、UTF-8 (BOM 有無) / Shift_JIS 自動判定 (`reader_load_text` コマンド)。
- 長文対策: 文境界優先・最大 120 トークンのチャンク分割。Irodori 絵文字アノテーションはどの分割経路でも分断しない。
- 合成はチャンクごとに既存 `synthesize_voice` を使用 (前処理・絵文字保護・フォールバックが適用)。再生は先読み 1 のパイプラインで、失敗チャンクはスキップして件数表示。チャンク間に 0.5 秒のポーズを入れて文の切れ目で一息つかせる。
- 読み上げ中は自発発話を抑制 (`set_reading_active` コマンド → 静音判定に追加)。リマインダーの静音特例は従来通り鳴る。停止ボタン / パネルクローズで即停止。

### 修正

- **実モデル初回発話クラッシュ 2 件**:
  - Irodori サイドカーの listen 準備完了前に合成リクエストを送る race を修正。
  - `ensure_engine` の二重初期化 TOCTOU (同時発話で init が競合) を修正。
- **Irodori V3 絵文字アノテーションの保持**: 漢字→ひらがな前処理 (voicevox OpenJTalk 経由) が絵文字アノテーションを落としていた問題を修正し、前処理後も絵文字による演出指示が残るようにした。

### ドキュメント

- **エンドユーザー向け取扱説明書 `docs/manual.md` を追加**。テキスト読み上げを含む全機能の利用手順を平易な日本語でまとめた。

### 開発向け (配布物に影響なし)

- dev サーバのポートを 5173 → 5273 に変更 (開発起動時のみ。配布インストーラには無関係)。

## 既知の制限 (このリリースでは対応しない)

v0.1.0 の「既知の制限」を引き継ぐ。加えて本リリース固有:

1. **テキスト読み上げは台本形式に未対応**: 話者指定・行ごとの速度・caption 演出は将来課題 (docs/text-reader-spec.md §7)。現状はメインキャラ 1 声で先頭から通し読みする。
2. **Irodori-TTS は引き続き β 扱い**: upstream 追従構成のため API 変更で不整合が起きうる。問題発生時は voicevox に自動フォールバックする。

その他 v0.1.0 リリースノート「既知の制限」2〜6 (advanced 独り言への時事ネタ混入 / リマインダー自然言語抽出の制限 / STT・自動更新・マルチ言語 / マルチモニタ DPI / 自動更新なし) はそのまま有効。

## 動作要件

v0.1.0 と同一。

- Windows 10/11 x64
- 一般機能: 任意の x86_64 CPU、3 GB 以上の空き RAM
- voicevox_core (推奨): 数百 MB のディスク + 一般的な CPU
- Irodori-TTS (任意): NVIDIA GPU (CUDA 12.8 対応、VRAM 4 GB 以上)、10 GB 以上のディスク (Python ランタイム ~5 GB + HF モデル ~4.3 GB)

## アップグレード手順

v0.1.0 からのアップグレードは、新しいインストーラを上書き実行するだけでよい (currentUser モード、DB スキーマ変更なし)。`%APPDATA%\ugg\` の既存データ (会話履歴・設定・TTS 資産) はそのまま引き継がれる。

v0.0.3 プロトタイプからの DB マイグレーションは提供しない。
