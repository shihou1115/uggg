## 概要

`C:\claude\ugga` での v0.0.3 プロトタイピング (2026-06-18) を起点に、仕様再定義 + 本開発を経て **M0〜M5 の機能を実装した最初のリリース**。Windows 10/11 x64 向け NSIS インストーラを `currentUser` モードで配布。

## 含まれる機能 (マイルストーン別)

### M0 起動骨格
- Tauri 2 透過ウインドウ、`%APPDATA%\ugg\` 配下にデータ配置
- ghost/shell.json パース失敗時の赤帯エラー表示

### M1 low モード対話
- 辞書 v3 (input_match / fallback / recall / monologue / events / system_messages)
- クリック透過 (8px グリッド + 50ms ポーリング)
- 起動挨拶 (時間帯別)、タイプライター (`talk_speed`)

### M2 advanced モード (LLM)
- OpenAI 互換クライアント (公式 / LM Studio / Ollama)
- コスト上限 (80% 警告 / 100% 自動降格)
- 連続 API エラーで一時降格 (5 分後復帰)
- user_profile 自動抽出 (origin=auto、上限 200 件)
- system_messages: cost_warning_80 / cost_limit_exceeded / mode_degraded / mode_recovered

### M3 存在感
- ランダムトーク (既定 10 分)、放置反応 (30 分、一度限り)、終了挨拶
- ポモドーロ (集中 25 / 休憩 5 / 4 ラウンド既定)、状態機械 + バッジ UI
- 静音モード (手動 + ポモドーロ集中時自動 + フルスクリーン自動)
- タスクトレイ + コンテキストメニュー、撫で/つつき (縦のみ部位判定)

### M4a/b voicevox_core 埋め込み + 漢字→ひらがな前処理
- 公式 C API 0.16.4 を libloading で動的ロード
- 規約同意付き自動 DL + GitHub PAT (keyring 保存)
- クレジット「VOICEVOX:話者名」常時表示
- 事前 init で初発話のラグ解消

### M4c Irodori-TTS Python サイドカー (β)
- DXGI で NVIDIA GPU 検出
- Python 3.11.9 embeddable + pip + PyTorch CUDA 12.1 + 関連ライブラリの初回 DL
- 自前 FastAPI サイドカー (`%APPDATA%\ugg\irodori\sidecar.py`)
- モックモード (正弦波 wav) で通信検証経路を確認可能
- アイドル 5 分で自動 shutdown、30 秒間隔のヘルスチェック、3 連続失敗で再起動
- 「実モデルを使う (β)」トグル — **実 Aratako/Irodori-TTS 結線は実機 GPU で TODO** (docs/quality_checklist.md G1〜G6 参照)

### M5 補助機能 (全 8 機能)
- M5-G: チャットログパネル (新しい順 200 件)
- M5-E: データエクスポート + 履歴クリア (JSON、`%USERPROFILE%\Downloads`)
- M5-F: ゴースト/シェル切替 UI (再起動で反映)
- M5-H: 自動起動 (`tauri-plugin-autostart`)
- M5-D: 更新通知 (update_feed_url、24h おき + 即時チェック、`update_notice_seen:<ver>` で重複防止)
- M5-C: 時事ネタ RSS (Google News + quick-xml + 暗い見出しフィルタ + advanced system prompt 注入用キャッシュ)
- M5-A: DnD ゴースト/シェル展開 (zip / フォルダ、zip slip 対策、1GB 上限、拡張子フィルタ、上書き確認 UX)
- M5-B: ツール (「N 分後」リマインダー / 時刻注入 / 📋 クリップボード補助、静音中も鳴る特例)

## 既知の制限 (このリリースでは対応しない)

1. **Irodori-TTS の実モデル推論コードは TODO**: `sidecar.py::RealModelBackend.synthesize` と `generate_voice_ref` が `NotImplementedError`。実機 GPU 環境で Aratako/Irodori-TTS 最新サンプルを参照しながら結線が必要。それまでは `tts_irodori_use_real_model=false` (既定) でモック wav が返る。
2. **advanced 独り言への時事ネタ混入は将来課題**: RSS キャッシュは溜まるが、`dialogue/monologue.rs` 経路は未統合。
3. **リマインダーの自然言語抽出は「N 分後 / 時間後 / 秒後」限定**: 「明日の朝 9 時」のような曖昧表現は未対応 (将来 LLM 経由の構造化抽出で対応予定)。
4. **STT / 自動更新 / マルチ言語 / 多話者キャプション**: spec §6 将来課題のとおり、本リリース範囲外。
5. **マルチモニタの DPI 差・起動モニタ選択**: 細やかな対応は将来課題。
6. **自動更新は無し**: `update_feed_url` 経由のゴースト告知のみ。手動 DL & 再インストール前提 (コード署名証明書なし)。

## 動作要件

- Windows 10/11 x64
- 一般機能: 任意の x86_64 CPU、3 GB 以上の空き RAM
- voicevox_core (推奨): 数百 MB のディスク + 一般的な CPU
- Irodori-TTS (任意): NVIDIA GPU (CUDA 12.x 対応、VRAM 4 GB 以上)、6 GB 以上のディスク

## アップグレード手順

新規インストールのみ。v0.0.3 プロトタイプからの DB マイグレーションは提供しない (本開発は新規 DB)。古い `%APPDATA%\ugg\` がある場合は事前にバックアップ → 削除を推奨。
