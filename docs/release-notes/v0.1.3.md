## 概要

v0.1.2 (2026-07-05) に続くマイナーリリース。**クリーンインストール環境で Irodori-TTS を有効化できなくなるデッドロックの修正**が中心。加えて取扱説明書 (manual.md) をインストーラに同梱し、初回起動時に一度だけ自動で開くようにした。DB スキーマに破壊的変更はなく、Tauri コマンドの外部契約にも変更はない (app_settings に `manual_shown` フラグを 1 件追加するのみ)。

## 変更点

### 修正: クリーンインストールで Irodori-TTS を選べないデッドロック

- v0.1.2 で入れた設定・音声タブのエンジン別表示 (VOICEVOX 選択時は Irodori 関連 UI を隠す) が、資産未ダウンロード時に Irodori 選択肢を無効化する既存ロジックと衝突していた。**資産を何も導入していないクリーンインストール環境**では、voicevox を選んでいる間 Irodori セクションごと隠れ、かつ Irodori に切り替える導線も無効化されるため、**Irodori 有効化のためのダウンロードボタンに一切到達できない**デッドロックが発生していた。
- **資産未導入のうちは、voicevox 選択中でも Irodori セクション (GPU 検出・ランタイムのダウンロード導線) を表示する**よう修正。ダウンロード導線を常に残すことで、クリーンインストール直後からでも Irodori-TTS を有効化できるようにした。資産導入後は従来どおりエンジン別表示に従う。

### 新機能: 取扱説明書の同梱と初回起動時の自動オープン

- 取扱説明書 `manual.md` を**インストーラに同梱**し、インストール先 (`%LOCALAPPDATA%\ugg\`) に `manual.md` として展開するようにした (`bundle.resources` に `../docs/manual.md` → `manual.md` を追加)。
- **初回起動時に一度だけ、OS 標準の関連付けで manual.md を自動で開く**。開いたかどうかは app_settings の `manual_shown` フラグで管理し、2 回目以降は開かない。**既存ユーザーが本版へ更新した直後も 1 度だけ開く** (フラグ未設定のため)。
- manual.md が見つからない場合はフラグを立てず何もしない (将来のビルドで開けるよう温存)。ユーザーはいつでも `%LOCALAPPDATA%\ugg\manual.md` を開いて読み返せる。

### ドキュメント

- **取扱説明書 `docs/manual.md` の拡充**: クイックスタート (最短手順) の追加、困ったときの FAQ 化、用語集の追加、各章へのスクリーンショット挿入位置の明示を行い、初見のユーザーが迷わず使い始められる構成にした。

## 配布物の変更

- `bundle.resources` に `../docs/manual.md` → `manual.md` のマッピングを追加。インストーラが取扱説明書をインストール先に同梱する。インストーラのサイズはこの分だけ増える。

## 既知の制限 (このリリースでは対応しない)

v0.1.2 の「既知の制限」をそのまま引き継ぐ。

1. **台本形式の将来課題** (docs/script-reader-spec.md §7): 話者は main / sub の 2 声まで / 話者マッピングは `slot` キー規約のみ (UI なし) / 行速度は playbackRate 適用 (ピッチ変動) / caption 以外の条件付け・台本のプレイリスト化は未対応。
2. v0.1.1 の「Irodori-TTS は引き続き β 扱い」、および v0.1.0 リリースノート「既知の制限」(advanced 独り言への時事ネタ混入 / リマインダー自然言語抽出の制限 / STT・自動更新・マルチ言語 / マルチモニタ DPI / 自動更新なし) はそのまま有効。

## 動作要件

v0.1.2 と同一。

- Windows 10/11 x64
- 一般機能: 任意の x86_64 CPU、3 GB 以上の空き RAM
- voicevox_core (推奨): 数百 MB のディスク + 一般的な CPU
- Irodori-TTS (任意): NVIDIA GPU (CUDA 12.8 対応、VRAM 4 GB 以上)、10 GB 以上のディスク (Python ランタイム ~5 GB + HF モデル ~4.3 GB)

## アップグレード手順

v0.1.2 / v0.1.1 / v0.1.0 からのアップグレードは、新しいインストーラを上書き実行するだけでよい (currentUser モード、DB スキーマ変更なし)。`%APPDATA%\ugg\` の既存データ (会話履歴・設定・TTS 資産) はそのまま引き継がれる。更新後の初回起動時に取扱説明書が一度だけ開く。

v0.0.3 プロトタイプからの DB マイグレーションは提供しない。
