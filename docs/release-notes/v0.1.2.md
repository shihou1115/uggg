## 概要

v0.1.1 (2026-07-04) に続くマイナーリリース。**テキスト読み上げツールの台本形式対応**が中心。加えて掛け合いの音声重なり不具合の修正と、設定・音声タブのエンジン別表示による UI 整理を含む。DB スキーマに破壊的変更はない。フロント内部コマンド `reader_load_text` の戻り値型を拡張したが、配布物 (インストーラ) の外部契約・設定・DB には影響しない。

## 変更点

### 新機能: テキスト読み上げの台本形式対応 (spec §4.5.8 / docs/script-reader-spec.md)

- 読み上げパネル表示中に **`.md` 台本ファイル** を DnD すると、話者を main / sub に振り分けて掛け合いで読み上げる。`.txt` は従来どおりメインキャラ 1 声のプレーン読み (完全互換)。
- 台本は upstream Irodori-TTS 形式: Markdown 内に **必須の ` ```json speakers ` / ` ```json lines `** と**任意の ` ```json defaults `** コードブロックを記述する。`speakers` の各話者 ID を `slot` キー (`"main"` / `"sub"`) で ugg の声にマッピングする。
- 行ごとに次を制御できる:
  - **speed** (速度オフセット、範囲 [-1.0, +1.0]): 設定の速度に合成して再生 (両エンジン一律)。
  - **caption** (演技指示、最大 200 文字): Irodori 実モデル時のみ有効。voicevox / フォールバック時はエラーにせず読み進め、パネルに注記を常時表示する。
  - **pause_after** (行の後の間、秒、範囲 [0.0, 10.0])。
- **不正な台本はロード時に全行検証し fail-fast** (部分再生しない)。未定義話者・`ref_wav` 指定・JSON 破損・範囲外の値・通常の Markdown 文書 (台本ブロック無し) などは、種別ごとの文言で原因を提示して再生を開始しない。
- Irodori 実モデル時は、台本が使う slot の参照音声が未生成なら**再生開始前に**検証してエラー表示する (片側の声が丸ごと無音になる前に止める)。
- 停止・クローズ・自発発話抑制・失敗チャンクスキップ・進捗表示は `.txt` 読みと共通。
- 書き方は docs/manual.md、サンプルは docs/samples/script-sample.md、詳細仕様は docs/script-reader-spec.md。

### 修正

- **掛け合いの音声重なり**: 雑談・つつき・撫で・チャット応答で、main の発声が完了する前に sub が話し始めて音声が重なる不具合を修正。発声キューを **main/sub を問わない全 slot 直列の単一キュー**に統一し、再生中に次アイテムの合成を先行させる (先読み 1) 構成にした。話者交代の無音は最小のまま、重なりを解消している。

### UI

- **設定・音声タブのエンジン別表示**: エンジンで VOICEVOX を選択時は Irodori-TTS 関連 UI を、Irodori-TTS を選択時は VOICEVOX の声 (style ID) 選択を非表示にした。現在のエンジンに関係のない項目を隠し、迷わない UI にした。

## 既知の制限 (このリリースでは対応しない)

v0.1.1 の「既知の制限」を引き継ぐ。加えて台本形式に固有の将来課題 (docs/script-reader-spec.md §7):

1. **話者は main / sub の 2 声まで**: 第 3 スロット以降には対応しない。voice_ref 層は任意 slot 対応済みだが、コマンド層の main/sub 固定の解除と管理 UI が別途必要。
2. **話者マッピングは `slot` キー規約のみ (UI なし)**: 話者 ID とスロットの対応は台本内の `speakers` ブロックで指定する。GUI でのマッピング編集はない。
3. **行速度は再生側 playbackRate で適用**: Irodori 実モデルの `duration_scale` 配線 (ピッチ非破壊の速度変更) は将来課題。playbackRate 方式では速度変更に伴いピッチが変動する。
4. **caption 以外の条件付け・台本のプレイリスト化**: `cfg_scale_caption` の行指定、defaults の他パラメタ透過、複数台本のキュー化は未対応 (複数 DnD は先頭 1 件のまま)。

v0.1.1 の「Irodori-TTS は引き続き β 扱い」、および v0.1.0 リリースノート「既知の制限」(advanced 独り言への時事ネタ混入 / リマインダー自然言語抽出の制限 / STT・自動更新・マルチ言語 / マルチモニタ DPI / 自動更新なし) はそのまま有効。

## 動作要件

v0.1.1 と同一。

- Windows 10/11 x64
- 一般機能: 任意の x86_64 CPU、3 GB 以上の空き RAM
- voicevox_core (推奨): 数百 MB のディスク + 一般的な CPU
- Irodori-TTS (任意): NVIDIA GPU (CUDA 12.8 対応、VRAM 4 GB 以上)、10 GB 以上のディスク (Python ランタイム ~5 GB + HF モデル ~4.3 GB)

## アップグレード手順

v0.1.1 / v0.1.0 からのアップグレードは、新しいインストーラを上書き実行するだけでよい (currentUser モード、DB スキーマ変更なし)。`%APPDATA%\ugg\` の既存データ (会話履歴・設定・TTS 資産) はそのまま引き継がれる。

v0.0.3 プロトタイプからの DB マイグレーションは提供しない。
