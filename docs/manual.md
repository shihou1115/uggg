# ugg 取扱説明書

ugg（うぐぐ）は、デスクトップの片隅に住みつくキャラクター（マスコット）です。
メインとサブの 2 体が透明なウインドウの上で暮らし、話しかけると返事をし、放っておくと
独り言をつぶやきます。Windows 10 / 11 (64bit) 専用です。

このマニュアルは v0.3.0 時点の内容です。

## クイックスタート（最短で使ってみる）

細かいことは後回しで、まず動かしてみたい人向けの最短手順です。各ステップの詳しい説明は
リンク先の章にあります。

1. **インストールして起動する** — `ugg_x.y.z_x64-setup.exe` を実行し、できた
   ショートカットから起動します。画面にキャラクターが現れます（→[1 章](#1-インストールと起動)）。
2. **初回の質問に答える（任意）** — 呼び名や話し方の希望を入れます。スキップしても
   かまいません（→[2 章](#2-初回セットアップ)）。
3. **話しかけてみる** — キャラクターの絵を **1 回クリック**すると文字入力欄が開きます。
   何か打ち込んで送信すると返事が返ってきます（→[3 章](#3-基本の操作)・[4 章](#4-会話について)）。
4. **声を付ける（任意）** — 設定 →「音声」ページで「声で話す」をオンにし、VOICEVOX の
   資産をダウンロードすると、キャラが声で話すようになります（→[5 章](#5-声で話させる音声合成)）。
5. **邪魔なときは隠す** — 右クリック →「ウインドウを隠す」で消せます。画面右下の
   トレイアイコンを左クリックで戻せます（→[3 章](#3-基本の操作)）。

> 📷 **スクリーンショット**: 起動直後、デスクトップにキャラクターが表示された状態

はじめから丁寧に読みたい場合は、次の「1. インストールと起動」から順にどうぞ。

## 1. インストールと起動

1. インストーラ `ugg_0.3.0_x64-setup.exe` を実行します。管理者権限は不要で、
   あなたのユーザー領域にインストールされます。
2. インストールが終わると、デスクトップとスタートメニューにショートカットができます。
3. ショートカットから ugg を起動すると、画面にキャラクターが現れます。

初めて起動したときは、この取扱説明書が一度だけ自動で開きます（次回以降は開きません）。
あとで読み返したいときは、いつでも `%LOCALAPPDATA%\ugg\manual.md`
（`C:\Users\ユーザー名\AppData\Local\ugg\manual.md`）を開いてください。

ウインドウには枠やタイトルバーがなく、キャラクターの絵の部分だけが表示されます。
キャラクター以外の透明な部分はクリックがそのまま下のアプリに通り抜けるので、
ugg を出したまま普通に作業できます。

## 2. 初回セットアップ

初めて起動すると「はじめまして」という小さな画面が出ます。いずれも任意で、
あとから設定でいつでも変えられます。

- **呼び名**: キャラクターにあなたをどう呼んでほしいか（例: しほ）。
- **話し方の希望**: フランクに、敬語で、など。
- **ネットの話題を会話に取り入れる**: オンにすると、後述の advanced モードのときに
  ニュースの話題を会話にまぜます。

「はじめる」で確定、「スキップ」で何も入力せずに閉じます。

> 📷 **スクリーンショット**: 初回セットアップ「はじめまして」画面（呼び名・話し方の入力欄）

## 3. 基本の操作

キャラクターの絵の上で、次の操作ができます。透明な部分では反応しません。

| 操作 | 反応 |
|---|---|
| **1 回クリック** | クリックしたキャラが「なにか用かな？」と返事をして、文字入力欄が開きます（もう一度クリックか Esc で閉じる）。ここから話しかけられます |
| **すばやく 2〜3 回クリック** | 「つつき」反応。触った高さ（頭・体など）で反応が変わります |
| **すばやく 4 回以上クリック** | 「たくさんつつかれた」反応 |
| **キャラの上でマウスを左右に往復** | 「撫で」反応（ボタンは押さずにカーソルを動かします） |
| **ドラッグ（押したまま動かす）** | キャラを左右に移動できます（メインとサブは別々に動かせます。高さはタスクバーの上で固定）。位置は次回起動時も覚えています |
| **右クリック** | キャラが返事をして、吹き出しの中にメニューが開きます（下記） |

つつき・撫で・クリックの返事は、メイン・サブどちらのキャラでも反応します。

### 右クリックメニュー

メインキャラを右クリックすると、メインが「ご用件はどれかな？」と返事をして、
**同じ吹き出しの中に**メニューが表示されます。サブキャラを右クリックした場合は、
サブが「用事ならメインに頼んでくれ」と案内したあと、メインのメニューに切り替わります。
メニューは項目を選ぶか、Esc キーか、メニューの外をクリックすると閉じます。

- **モード切替**（low ⇔ advanced） — 会話モードを切り替えます（[4 章](#4-会話について)参照）
- **静音モード ON / 解除** — 自発的な発話（独り言など）を止める・戻す
- **予定・ToDo** — リマインダー・やることリスト・今日明日の予定のパネルを開きます（[6 章](#6-便利な機能)参照）
- **ポモドーロタイマー** — 集中タイマーの操作パネルを開きます（[6 章](#6-便利な機能)参照）
- **テキスト読み上げ** — .txt ファイルの朗読パネルを開きます（[6 章](#6-便利な機能)参照）
- **設定を開く** — 設定画面を開きます
- **ウインドウを隠す** — キャラを画面から隠します（タスクトレイから戻せます）
- **終了** — ugg を終了します

> 📷 **スクリーンショット**: キャラクター上で右クリックしてメニューが開いた状態

### タスクトレイ

画面右下の通知領域に ugg のアイコンが常駐します。

- **アイコンを左クリック** — ウインドウの表示 / 非表示を切り替え
- **アイコンを右クリック** — 表示/非表示・モード切替・静音・設定を開く・終了

## 4. 会話について

ugg には 2 つの会話モードがあります。設定画面か右クリックメニューで切り替えます。

### low モード（辞書・無料・オフライン）

あらかじめ用意された辞書にもとづいて返事をします。インターネットや API キーは不要で、
完全に無料・オフラインで動きます。決まったパターンの受け答えが中心です。

### advanced モード（AI 会話）

大規模言語モデル（LLM）を使って、より自由な会話をします。利用には LLM の接続先設定が
必要です（[7 章の AI・拡張ページ](#ai拡張ページ)参照）。次のいずれかを使えます。

- **クラウドの API**（OpenAI など・従量課金）
- **手元の PC で動く LLM**（LM Studio / Ollama など・無料）

advanced モードでは会話の中からあなたに関する情報を少しずつ覚えていきます。

### 話しかけ以外の発話

- **起動のあいさつ**: 立ち上げると時間帯に応じてあいさつします。
- **独り言**: しばらく操作がないと、ときどき自分から何かつぶやきます（間隔は設定で変更可、
  0 分で無効）。
- **放置反応**: 長く放っておくと 1 度だけ反応します。
- **状況に応じた声かけ（任意・初期はすべてオフ）**: 休憩促し・深夜利用の声かけ・
  バッテリー低下の知らせ・ToDo の思い出しなど。設定の「日常支援」ページで種類ごとに
  オンにできます（[6 章](#6-便利な機能)参照）。
- **朝・夜の定例会話（任意・初期はオフ）**: 1 日の始まりと終わりに、その日の予定・ToDo・
  天気などをまとめて話しかけます（[6 章](#6-便利な機能)参照）。

これらの自発発話は「**静音モード**」で止められます（右クリックメニューや設定から）。
設定の「日常支援」ページで**夜間静音**（時間帯を決めて自発発話を止める）も設定できます。
登録したリマインダーとカレンダーの予定通知だけは、静音中でも鳴ります。

## 5. 声で話させる（音声合成）

初期状態では文字だけで話しますが、設定で音声合成を有効にすると声が付きます。
2 種類のエンジンを選べます。設定画面の「音声」ページで切り替えます。エンジンを
選ぶと、そのエンジンに関係のない項目は自動で隠れます（VOICEVOX 選択時は
Irodori-TTS 関連の設定が、Irodori-TTS 選択時は VOICEVOX の話者選択が非表示に
なります）。今使っている項目だけが並ぶので、設定に迷いません。

> 📷 **スクリーンショット**: 設定「音声」ページ（VOICEVOX 選択時 / Irodori-TTS 選択時の見え方の違い）

### VOICEVOX（推奨・無料・CPU）

無料の音声合成エンジンです。GPU は不要で、普通のパソコンで動きます。

1. 設定 →「音声」ページ →「声で話す」をオンにします。
2. エンジンで「VOICEVOX」を選びます。
3. 「音声資産」の「資産をダウンロード」を押します（数百 MB のダウンロード）。
   ダウンロードすると VOICEVOX の利用規約に同意したものとみなされます。
4. ダウンロード後、メイン・サブそれぞれの声（話者）を選べるようになります。

声で話しているときは、画面に「VOICEVOX: 話者名」というクレジットが表示されます
（VOICEVOX の規約にもとづく表示です）。

### Irodori-TTS（GPU 必須・高品質）

NVIDIA GPU を積んだ PC 向けの、より自然で表情豊かな音声エンジンです。
「高品質モード」とも表示されます。**NVIDIA GPU（CUDA 12.x 対応・VRAM 4GB 以上）が必要**です。

1. 設定 →「音声」ページ →「音声 (Irodori-TTS / 高品質モード)」の欄を見ます。
2. 「GPU 検出」に、お使いの NVIDIA GPU 名が出ていれば使えます
   （出ていなければこの PC では使えません）。
3. 「ランタイムをダウンロード」を押します。**合計で 10GB 前後**のダウンロードと、
   10〜20 分ほどの時間がかかります（Python 実行環境・PyTorch・AI モデル本体を取得）。
4. ダウンロードが済むと「実モデルを使う (β)」が選べるようになります。これをオンにすると、
   AI が生成した声で話します。
5. **参照音声（声のもと）を作ります**。メイン／サブそれぞれに、声の雰囲気を言葉で
   書いて（例:「明るく元気な若い女性の声」）「生成」を押すと、その雰囲気の声が作られます。
   「プレビュー」で試聴、「削除」でやり直せます。

Irodori-TTS は β（試験的）機能です。うまく動かないときは自動で VOICEVOX に切り替わり、
キャラが「いつもの声で話すね」と教えてくれます。

#### 絵文字で声の演出を変える（Irodori-TTS のみ）

Irodori-TTS で読み上げるテキストに特定の絵文字を入れると、声の感情や演出が変わります。
絵文字そのものは読み上げられず、演出の指示として働きます。同じ絵文字を重ねると効果が
強まります。効果は完璧ではなく、雰囲気が変わる程度のこともあります。

例:
- 「😊今日はいい天気だね」 → 楽しげに
- 「うぅ…😭ひどいよ…😭」 → 泣きそうな声で
- 「👂ここだけの話なんだけど」 → ささやき声で
- 「🤧🤧ごめん、風邪引いちゃって」 → 鼻声・咳き込みまじりで

主な絵文字: 😊 楽しげ / 😭 泣き / 😠 怒り / 😱 悲鳴 / 🥺 震え声 / 👂 ささやき /
😮‍💨 ため息 / ⏸️ 間 / ⏩ 早口 / 🐢 ゆっくり / 🤭 くすくす笑い / 😴 寝言 / 🎵 鼻歌 /
📖 ナレーション調 など全 45 種（対応外の絵文字は無視されます）。

VOICEVOX では絵文字による演出はできません（絵文字は読み飛ばされます）。

## 6. 便利な機能

### ポモドーロタイマー

集中と休憩を繰り返す時間管理です。右クリックメニューの「ポモドーロタイマー」で専用パネルが開き、
そこから操作します。

- **開始** — タイマーを始めます。
- **停止** — カウントダウンを一時停止します。もう一度押すと同じ残り時間から**再開**します。
- **中断** — タイマーを取りやめて最初の状態に戻します。
- **閉じる** — パネルを閉じます（タイマーが動いていればそのまま続き、画面上部のバッジで表示されます。バッジをクリックすると再びパネルが開きます）。
- **時間設定** — 集中・休憩の分数とラウンド数をパネルで変えられます（設定パネルと同じ値で、保存されます）。進行中は変更できません。

集中時間中は自動で静音になります（集中中に一時停止しても、静かなままです）。

### リマインダー（自然な言葉で登録）

チャットで話しかけるだけで、指定の時間にキャラクターが知らせてくれます。AI を介さず
即座に登録されるので、low モードでも・オフラインでも使えます。時間になると静音中でも
知らせます。たとえば:

- 「**3 分後にお茶**」「**1 時間後に休憩**」（〜分後・〜時間後）
- 「**18 時に洗濯物**」「**明日の朝ゴミ出し**」「**7 月 20 日の 10 時に病院**」
- 「**毎日 9 時に薬**」「**毎朝ストレッチ**」「**毎週月曜 9 時にゴミ出し**」（繰り返し）

知らせたときに「あとで」にしたければ、パネルの**スヌーズ**で 10 分後にもう一度
鳴らせます。登録済みの一覧・完了・削除は、右クリックメニューの「**予定・ToDo**」
パネルから行います（行をクリックすると通知の履歴も見られます）。

### やることリスト（ToDo・日課）

厳密な時刻がない「やること」の管理です。右クリック → 「**予定・ToDo**」パネルの
ToDo 欄で使います。

- **今日 / 今週 / いつか** の 3 つのリストに分けて登録します。
- チェックで完了。キャラクターがひとこと労ってくれます。
- **★** で優先マーク、**↻** で日課（毎日 / 毎週）にできます。日課は完了しても
  翌日（翌週）に自動で復活します。
- 朝には「今日は 3 件あるよ」のように件数を教えてくれます。終了時に未完了が
  残っていると、終了のあいさつでそっと触れます。

### カレンダー（予定の読み込みと通知・任意）

お使いのカレンダーの予定（ICS 形式）を読み込んで、**開始前にキャラクターが
知らせてくれます**（初期設定では 15 分前、変更可）。予定の書き込みはしない
「読み取り専用」です。

1. 設定 → 「日常支援」ページの「カレンダー」で、ICS ファイルのパスか URL を追加
   します（Google カレンダー等からエクスポートした .ics ファイルが使えます）。
2. 「いま取得」を押すと読み込みます。以後は 30 分ごとに自動で更新されます。
3. 今日・明日の予定は「予定・ToDo」パネルに表示されます。終日の予定は朝 8 時に
   知らせます。

URL を登録した場合、その取得のときだけインターネットに接続します。オフラインの間は
最後に読み込んだ内容を表示します。

### 状況に応じた声かけと 🔕 ボタン

設定 → 「日常支援」ページでオンにすると（初期はすべてオフ）、状況に応じて
キャラクターが声をかけてくれます。

- **休憩促し**: PC を 90 分続けて使うごとに「ちょっと伸びでもしない？」
- **深夜利用**: 23 時〜5 時に使っていると、ひと晩に 1 回だけ声をかけます
- **バッテリー低下**: ノート PC で残り 15% を切ると知らせます
- **ToDo の思い出し**: 午後に未完了を思い出させ、夕方に何日も残っているものの
  整理を提案します

声かけが邪魔だったときは、**吹き出しの右上に出る 🔕 を押してください**。その種類の
声かけの頻度が下がり、3 回押すと自動でオフになります（設定からいつでも戻せます）。
集中タイマー・静音モード・夜間静音の間は声かけしません。

### 天気（生活情報）

お住まいの地域を登録すると、キャラクターが天気を会話に取り入れます（後述の
「朝・夜の定例会話」や「降雨の一言」で使われます）。初期状態ではオフで、地域を登録すると
有効になります。設定 →「日常支援」ページの「天気」で設定します。

1. 「地域を検索」に市区町村名や都道府県名を入れて「検索」を押します（例: 横浜市、渋谷）。
2. 出てきた候補から住んでいる地域を選ぶと、その地域が登録され、天気が有効になります。
   **地域を選んだ時点で、天気予報サービス（Open-Meteo）の利用に同意したものとみなします**。
   送信されるのは天気を取り寄せるための地名と座標だけで、**座標は市区町村単位
   （おおよその範囲）に丸めて**から送られます。
3. 「いま取得」を押すと、今日・明日の天気を取り寄せて表示し、正しく設定できたか確認できます。
4. やめたいときは「解除」を押します。登録した地域と取得済みの天気データが消え、天気は
   オフに戻ります。

見つからないときのコツ: 市区町村名・都道府県名で検索してください。表記によっては
見つからないことがあります（「東京」で出ないときは「東京都」、というように言い換えて
みてください）。

**降雨の一言**（初期はオフ）: オンにすると、その日に雨が見込まれる朝に一度だけ
「傘があると安心かも」のように知らせます。カレンダーにその日の外出予定（時刻のある予定）が
あると、言い回しが少し強くなります。これも「状況に応じた声かけ」の仲間なので、🔕 ボタンで
頻度を下げられます。

天気が有効な間は、画面下のクレジット表示に「天気: Open-Meteo.com」が出ます（天気データは
Open-Meteo.com による CC BY 4.0 ライセンスです。VOICEVOX のクレジットと同じく、利用規約に
もとづく表示です）。

### 朝・夜の定例会話

1 日の始まりと終わりに、その日の情報をまとめてキャラクターが話しかけてくれる機能です。
**初期状態ではオフ**で、設定 →「日常支援」ページの「朝・夜の定例会話」で、朝・夜それぞれを
個別にオンにします。

- **朝**（既定 8:00）: あいさつと、今日の予定・今日の ToDo の件数・未完了のリマインダー・
  今日の天気（天気が有効なとき）をまとめて話します。
- **夜**（既定 21:00）: ねぎらいと、今日終わった ToDo の数・持ち越した用事・明日の予定・
  明日の天気（天気が有効なとき）をまとめて話します。

朝・夜それぞれに **時刻** と **話す曜日** を設定できます。話題がないときはその項目を省き、
全部なくても短いあいさつだけはしてくれます。

- **話しかけるタイミング**: 「設定した時刻ちょうど」ではなく、**その時刻を過ぎてから、
  あなたが最初に PC を使っているとき**に 1 回だけ話しかけます（朝 8:00 に PC がついて
  いない日でも大丈夫です）。ただし**設定時刻から 6 時間を過ぎると、その日の分はお休み**
  します（夜遅くに「おはよう」と言い出さないための区切りです）。
- **朝の告知とのまとめ**: 朝の定例会話をオンにしている間は、これまで別々に出ていた
  「今日は ToDo が◯件あるよ」の告知と「降雨の一言」は、朝の定例会話の中にまとめられます
  （同じことを二重に言いません）。
- **夜間静音との関係**: 設定した時刻が夜間静音の時間帯に入っていると、その枠は
  話しかけられません。設定画面がこの重なりを見つけて警告を出します。
- **邪魔なとき**: 話しかけが邪魔なときは吹き出しの 🔕 を押してください。3 回押すと、
  その枠（朝または夜）が自動でオフになります（設定からいつでも戻せます）。
- **advanced モードのとき**: 話す中身は同じですが、言い回しが会話らしく整えられます
  （うまくいかないときは決まった文章でそのまま話します）。

### 時事ネタ（ニュースの話題）

「ツール」や「興味分野」を有効にすると、advanced モードのときにニュースの話題を会話に
まぜます。興味のあるキーワード（例: アニメ, Rust, 宇宙開発）を設定しておくと、それに
沿った話題を拾います。暗い見出し（事件・災害など）は自動で避けます。

### クリップボード貼り付け

チャット入力欄の 📋 ボタンを押すと、コピー中の文章を入力欄に貼り付けられます。
📋 ボタンは設定の「ツールを有効化」がオンのときだけ表示されます。

### チャットログ

過去のやりとりを一覧で見られます。設定画面の「システム」ページの「ログを開く」から
開けます。

### テキスト読み上げ

テキストファイル (.txt) をキャラの声で朗読させられます。

1. キャラを右クリック → 「テキスト読み上げ」でパネルを開きます。
2. パネルを開いたまま、.txt ファイルを ugg のウインドウにドラッグ&ドロップします。
3. メインキャラの声で先頭から読み上げます。パネルに進捗と今読んでいる箇所が出ます。
4. 「停止」ボタン、またはパネルを閉じると止まります。

> 📷 **スクリーンショット**: テキスト読み上げパネル（進捗「n / m」と現在読んでいる箇所の表示）

- 文字コードは UTF-8 と Shift_JIS に対応しています (最大 1MB)。
- 声・速度・音量は設定の「音声」ページに従います。
- Irodori-TTS (実モデル) のときは、テキスト中の絵文字による演出
  ([絵文字で声の演出を変える](#絵文字で声の演出を変えるirodori-tts-のみ)) も有効です。
- 読み上げ中、キャラは独り言を控えます (リマインダーは鳴ります)。

#### 台本形式で読み上げる（上級者向け）

台本ファイル (.md) をドラッグ&ドロップすると、メインとサブの 2 体の声で
台本どおりに掛け合い読み上げができます。行ごとに話者・速度・間を指定でき、
朗読劇のような読み上げが作れます。

台本は Markdown ファイルの中に、次の 3 つのコードブロックを書きます。

```` markdown
```json speakers
{ "host": { "slot": "main" }, "guest": { "slot": "sub" } }
```
```json lines
[
  { "speaker": "host",  "text": "ねえ、聞いた？" },
  { "speaker": "guest", "text": "知ってるわ。", "pause_after": 0.6, "speed": 0.1 },
  { "speaker": "host",  "text": "えええ！！", "caption": "驚いて大声で" }
]
```
```json defaults
{ "default_pause_seconds": 0.3 }
```
````

（動くサンプルは [docs/samples/script-sample.md](samples/script-sample.md) にあります）

- **`speakers`（必須）**: 台本に登場する話者を決めます。話者の名前（`host` など、
  自由に付けられます）ごとに `slot` を指定し、`"main"`（メインキャラの声）か
  `"sub"`（サブキャラの声）のどちらで読むかを割り当てます。外部の音声ファイル
  （ref_wav）を指定することはできません。
- **`lines`（必須）**: 読み上げる行を上から順に並べます。各行のキーは次のとおりです。
  - `speaker`（必須）: どの話者が読むか。`speakers` で決めた名前を書きます。
  - `text`（必須）: 読み上げる文章。
  - `speed`（任意）: その行だけの速度。`-1.0`（遅い）〜`+1.0`（速い）。
  - `caption`（任意）: 演技の指示（例:「驚いて大声で」）。200 文字まで。
  - `pause_after`（任意）: その行を読んだ後の間（秒）。`0`〜`10`。
- **`defaults`（任意）**: 省略時の共通設定。`default_pause_seconds`（行後の間の
  既定値。省略すると 0.3 のようにまとめて指定できます）と `speed` を書けます。

**注意: `caption`（演技指示）が効くのは Irodori-TTS（実モデル）を使っているときだけ**
です。voicevox のときは `caption` は無視され、読み上げパネルに「※ 演出指示 (caption)
は現在の音声エンジンでは無視されます」という注記が出ます（不具合ではありません）。
話者の切り替え・速度・間は、どちらの音声エンジンでも有効です。

台本に書き間違い（`speakers` にない話者を使った、数値が範囲の外、JSON が壊れている、
`speakers` や `lines` ブロックが無い、など）があると、読み上げは**始まらず**、
パネルに原因が表示されます。表示された内容を直してから、もう一度ドラッグ&ドロップ
してください。

> 📷 **スクリーンショット**: 台本にエラーがあったときの表示例（原因のメッセージが出た状態）

## 7. 設定画面ガイド

設定画面は右クリックメニューかトレイの「設定を開く」で開きます。上部のプルダウンで
6 つのページを切り替えます。変更したら下の「保存」を押してください。

> 📷 **スクリーンショット**: 設定画面（上部のページ切り替えプルダウンと「保存」ボタン）

### 基本ページ

- **対話モード**: low（辞書）/ advanced（AI）の切り替え
- **表示スケール**: キャラの大きさ
- **タイプライター速度**: 文字が表示される速さ（遅い〜一気に）
- **OS ログイン時に自動起動**: Windows 起動時に ugg も立ち上げる

### キャラクターページ

- **ゴースト / シェル**: 使うキャラクターの中身（性格・辞書）と見た目を選びます
  （変更後はアプリの再起動で反映）
- **静音モード**: 自発的な発話を止める
- **フルスクリーンアプリ前面で自動静音**: 全画面のゲームや動画のときに自動で静かにする
- **独り言間隔**: 何分ごとに独り言を言うか（0 で無効）

### 日常支援ページ

[6 章](#6-便利な機能)の各機能のスイッチ類がまとまっています。

- **日常支援を有効化**: チャット自然文でのリマインダー登録の大元スイッチ
- **リマインダー通知**: 発火時にキャラクターが知らせるか（登録自体は常時可能）
- **状況に応じた声かけ**: 休憩促し・深夜・バッテリー・ToDo 思い出しの個別オン/オフと、
  声かけ同士の最低間隔
- **カレンダー**: ICS の URL / ファイルパスの追加・削除、「いま取得」、開始何分前に知らせるか
- **朝・夜の定例会話**: 朝・夜それぞれの有効/無効・時刻・話す曜日（[6 章](#6-便利な機能)参照）
- **天気**: 地域の検索・登録・解除、「いま取得」、降雨の一言のオン/オフ（[6 章](#6-便利な機能)参照）
- **夜間静音**: 指定した時間帯（例: 23:00〜7:00）は自発的な発話を止めます。
  リマインダーとカレンダーの通知は夜間静音より優先されます

### 音声ページ

[5 章](#5-声で話させる音声合成)を参照してください。VOICEVOX / Irodori-TTS の設定と、
話者・速度・音量の調整があります。

### AI・拡張ページ

- **LLM**: advanced モードの接続先（プロバイダ名・モデル・URL・API キー・月額上限）。
  月額上限を決めておくと、8 割で警告し、超えると自動で low モードに下がります。
- **ポモドーロ**: 集中・休憩の分数、ラウンド数
- **ツール**: advanced モードでの時刻補強と、入力欄の 📋 クリップボード貼り付けのオン/オフ
  （リマインダーは「日常支援」ページに移りました）
- **興味分野**: 時事ネタのキーワード設定と「いますぐ取得」

### システムページ

- **更新通知**: 新バージョンのお知らせ用フィード URL と「いますぐチェック」
- **データ管理**: チャットログを開く / データの書き出し（JSON エクスポート）/ 履歴クリア。
  「記憶（覚えた内容）も含める」にチェックすると、エクスポートやクリアの対象に含めます。

## 10. 困ったとき（FAQ）

### 起動・表示

**Q. キャラが出てこない / 起動してすぐ消える**
A. まずタスクトレイ（画面右下）に ugg のアイコンが残っていないか確認し、あれば左クリックで
表示できます。エラーの小窓が一瞬出た場合は、その内容を控えて開発元に伝えてください
（配布版は起動に失敗すると原因をこの窓に表示します）。

**Q. キャラを一時的に消したい**
A. 右クリック →「ウインドウを隠す」で消せます。トレイアイコンの左クリックで戻せます。
終了ではないので、裏で常駐したままです。

**Q. 透明な部分をクリックすると下のアプリに届くのはなぜ? キャラに当たらない**
A. ugg はキャラクターの絵がある部分だけを「触れる」ようにし、透明な部分の上に
カーソルがあるときはクリックを下のアプリへ通します。これは仕様で、ugg を出したまま
作業できるようにするためです。キャラに反応させたいときは、絵のある部分をクリック
してください。

**Q. キャラの位置がおかしくなった / 元に戻したい**
A. 位置はドラッグ（押したまま動かす）で自由に変えられ、次回起動時もその位置を覚えています。
専用の「位置リセット」ボタンはありません。画面外に行ってしまった等で見失った場合は、
ドラッグで見える位置へ動かしてください。

**Q. 複数モニタで表示位置がずれる / 意図しない画面に出る**
A. モニタごとの拡大率（DPI）が違う環境での細かな位置合わせは今のところ発展途上です。
ドラッグで使いたい画面へ移動してください。

### 声・音声合成

**Q. 声が出ない**
A. 設定 →「音声」ページで「声で話す」がオンか、音声資産をダウンロード済みかを確認して
ください。音量が 0 になっていないかも確認します。

**Q. Irodori-TTS（高品質モード）が使えない / 選べない**
A. NVIDIA GPU（CUDA 対応）が必要です。「GPU 検出」欄に GPU 名が出ていなければ、この PC
では使えません（VOICEVOX をお使いください）。ランタイムのダウンロードが未完了のあいだは
「実モデルを使う」は選べません。動作中に問題が起きると自動で VOICEVOX に切り替わります。
手動で戻すには、設定の音声エンジンを「VOICEVOX」に変えて保存してください。

**Q. 台本の caption（演技指示）が効かない**
A. caption が効くのは Irodori-TTS（実モデル）を使っているときだけです。VOICEVOX では
無視され、読み上げパネルに注記が出ます（不具合ではありません）。話者の切り替え・速度・間は
どちらのエンジンでも有効です（[6 章の台本形式](#台本形式で読み上げる上級者向け)参照）。

**Q. 読み上げが途中の文を飛ばした / 「N 件スキップ」と出た**
A. ある部分の音声合成に失敗したとき、その箇所だけ飛ばして続きを読み上げ、最後に
スキップした件数を知らせます。読み上げ自体は止まりません。Irodori-TTS で起きやすい場合は、
VOICEVOX に切り替えると安定することがあります。

### 会話

**Q. AI 会話（advanced）が返事をしない**
A. LLM の接続先（プロバイダ・モデル・URL・API キー）が正しいか確認してください。
また、月額上限の 8 割を超えると警告し、上限を超えると自動で low モード（辞書）に下がります。
設定 →「AI・拡張」ページで上限と現在の状況を確認してください。

**Q. 独り言や通知がうるさい**
A. 右クリックメニューか設定で「静音モード」をオンにしてください。独り言だけを止めたい
ときは、設定の「独り言間隔」を 0 にします。

**Q. 覚えられた内容を消したい**
A. 設定 →「システム」→「履歴クリア」で「記憶（覚えた内容）も含める」にチェックして
実行すると、キャラが覚えたあなたの情報も消せます（[9 章](#9-データの管理)参照）。

## 11. データの保存場所

- 会話履歴・設定など: `%APPDATA%\ugg\`（`C:\Users\ユーザー名\AppData\Roaming\ugg\`）
- 音声（VOICEVOX）の資産: `%APPDATA%\ugg\voicevox\`
- 高品質モード（Irodori-TTS）の実行環境とモデル: `%APPDATA%\ugg\irodori\`
- アプリ本体: `%LOCALAPPDATA%\ugg\`（`C:\Users\ユーザー名\AppData\Local\ugg\`）

アンインストールはスタートメニューの ugg のアンインストール、または
`%LOCALAPPDATA%\ugg\uninstall.exe` から行えます。会話データ（`%APPDATA%\ugg\`）は
残るので、完全に消したい場合は手動で削除してください。

## 12. 用語集

このマニュアルに出てくる ugg 特有の言葉をまとめます。

- **ゴースト** — キャラクターの「中身」。性格・話し方・返事の辞書などをまとめたものです。
  同じ見た目でもゴーストを変えれば性格が変わります。
- **シェル** — キャラクターの「見た目」。各ポーズの絵をまとめたものです。ゴーストと
  シェルの組み合わせで 1 体のキャラクターになります。
- **メイン / サブ** — ugg は 2 体で暮らします。手前で主に話すのが「メイン」、掛け合いの
  相手が「サブ」です。撫でやつつきに反応するのはメインだけです。
- **スロット（slot）** — 声の枠のこと。「main」がメインキャラの声、「sub」がサブキャラの
  声を指します。台本形式では、どちらの声で読むかをこの名前で指定します。
- **low モード** — 辞書にもとづいて返事をする会話モード。無料・オフラインで動きます。
- **advanced モード** — LLM（AI）を使ってより自由に会話するモード。接続先の設定が必要です。
- **LLM** — 大規模言語モデル。advanced モードの会話を生む AI のことです。クラウドの API
  でも、手元の PC で動くものでも使えます。
- **参照音声（声のもと）** — Irodori-TTS で使う、声の雰囲気の見本。言葉で雰囲気を書いて
  生成し、メイン・サブそれぞれに設定します。
- **caption（キャプション／演技指示）** — 台本形式で、その行をどう演じるか（例:「驚いて
  大声で」）を指定するもの。Irodori-TTS（実モデル）のときだけ効きます。
- **台本形式** — メインとサブの掛け合いを行ごとに指定して読み上げさせる、`.md` ファイルの
  書式。話者・速度・間・演技指示を指定できます（[6 章](#台本形式で読み上げる上級者向け)参照）。
- **静音モード** — 独り言などの自発的な発話を止める状態。右クリックメニューや設定から
  切り替えます。リマインダーの通知は静音中でも鳴ります。
- **ポモドーロ** — 集中と休憩を繰り返す時間管理法。ugg にタイマー機能があります。
- **時事ネタ** — advanced モードのときに会話へまぜるニュースの話題。興味分野の
  キーワードに沿って集めます。
- **ツール** — リマインダー・時刻・クリップボード貼り付けなどの補助機能。設定でまとめて
  オン/オフします。
